**MCP (Model Context Protocol)** は、Lovable のような AI システムを外部のツール、サービス、データソースに接続するためのオープン標準です。

Lovable を Notion や Atlassian、Linear などのツールに接続すると、Lovable Agent はチーム固有のコンテキストを利用して、より正確なコード、プロトタイプ、ドキュメントを生成できるようになります。

MCP サーバーを接続すると、Lovable は構造化データ（ドキュメント本文や課題、ダイアグラムのテキストなど）を読み取り、対応している場合は項目の作成や更新といった限定的なアクションを実行できます。

Lovable には、すべてのプランで利用できる複数の事前構成済み MCP サーバーが含まれており、Business および Enterprise プランではカスタム MCP サーバーを追加することもできます。

<div id="why-connect-mcp-servers">
  ## なぜ MCP サーバーを接続するのか
</div>

Lovable に MCP サーバーを接続すると、次の 2 つの重要な機能が使えるようになります。

- **既存のコンテキストを Lovable に取り込む。**\
  Lovable Agent は、チームのドキュメント、チケット、デザインファイルを読み取り、チームの標準やルールに沿ったプロトタイプやフローを構築できます。
- **Lovable アプリをあなたのワークフローに接続する。**\
  Lovable Agent は、接続されたツール上で後続の処理を実行できます。たとえば、チケットのステータス更新、コメントとしてのプロトタイプリンクの追加、対応しているツールであれば新規アイテムの作成などです。

Lovable があなたのチームの世界を理解することで、手探りの作業が減り、イテレーションが加速し、**アイデアからライブアプリまで今までにない速さで到達できる**ようになります。

<div id="how-to-connect-a-prebuilt-mcp-server">
  ### 事前構築済み MCP サーバーを接続する方法
</div>

一般的なサードパーティ製アプリやサービス向けに用意されたインテグレーションを使うには、次の手順を実行します。

1. **Settings → Integrations → Your MCP servers** に移動します。
2. **MCP サーバーを選択** します（例：Notion、Linear、Atlassian）。
3. **Connect** をクリックします。サインインし、Lovable にあなたのアカウントへのアクセスを許可します。

これらの手順に加えて、[n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server) で次の操作を行う必要があります。

- **Settings → MCP access** に移動し、**Enable MCP access** のトグルをオンにします（インスタンスのオーナーまたは管理者権限が必要です）
- 公開したい各ワークフローについて、ワークフローエディタの **Settings** に移動し、**Available in MCP** のトグルをオンにします。

<div id="custom-mcp-servers-business-and-enterprise">
  ## カスタム MCP サーバー（Business および Enterprise）
</div>

最大限の柔軟性を得るために、**カスタム MCP サーバー**を接続して、社内システムやサードパーティシステムを Lovable に統合できます。プライベート API、社内 CRM、カスタムデータソースなどに最適です。

Lovable と互換性のある高品質なリモート MCP サーバーを厳選した一覧である [remote MCP servers](https://mcpservers.org/remote-mcp-servers) を参照してください。

<div id="how-to-connect-a-custom-mcp-server">
  ### カスタム MCP サーバーを接続する方法
</div>

カスタムまたは社内の MCP サーバーを接続するには、次の手順に従います。

1. **Settings → Integrations → Your MCP servers** に移動します。
2. **New MCP server** の横にある **Set up** をクリックします。
3. **Server name**: サーバーが分かりやすいような名前を入力します（例: _Internal CRM_ や _Analytics API_）。
4. **Server URL**: Lovable から MCP サーバーにアクセスできるアドレスを入力します。
5. **Authentication:**
   - **OAuth (default):** Lovable はデフォルトで OAuth を使用します。**Add & authorize** をクリックすると、認可画面が表示されます。
   - **Bearer token or API key:** OAuth をサポートしていないサーバーの場合は、ベアラートークンまたは APIキーで認証できます。
   - **No authentication**: サーバーが認証を必要としない場合は、認証情報なしで接続します。
6. **Add server** をクリックします。カスタム MCP サーバーが一覧に表示され、ビルド時のコンテキストデータを提供できるようになります。

<div id="security-and-permissions">
  ## セキュリティと権限
</div>

MCP サーバー接続は**ユーザー単位の個人用**です。**Settings → Integrations** から、いつでも接続を確認・取り消しできます。

プロジェクトレベルで素早くアクセスするには、プロンプトボックスの `+` をクリックして **Integrations** に進みます。

<img
  src="/images/mcp_integrations_project_access.png"
  alt="mcp_integrations_project_access.png"
  title="mcp_integrations_project_access.png"
  style={{ width:"51%" }}
/>

**Business** および **Enterprise** プランでは、ワークスペースの **admins と owners** が、すべてのユーザーに利用可能なサーバーを管理できます。

- **Settings → Privacy & security** に移動して、ワークスペース全体の **MCP server access**（MCP サーバーへのアクセス）を有効化または無効化します。
- **Settings → Integrations → Admin settings** に移動して、個々の MCP サーバーを有効化または無効化します。

**MCP (Model Context Protocol)** は、Lovable のような AI システムを外部のツール、サービス、データソースへ接続するためのオープンな標準仕様です。

Lovable を Notion、Atlassian、Linear などのツールに接続すると、Lovable Agent はチームの実際のコンテキストを利用して、より正確なコード、プロトタイプ、ドキュメントを生成できます。

MCP サーバーを接続すると、Lovable は構造化されたデータ（ドキュメント、課題、ダイアグラムのテキストなど）を読み取り、サーバーが対応している場合は、アイテムの作成や更新といった限定的なアクションを実行できます。

Lovable には、すべてのプランで利用できる複数のあらかじめ用意された MCP サーバーが含まれており、Business プランと Enterprise プランではカスタム MCP サーバーも追加できます。

<div id="why-connect-mcp-servers">
  ## MCP サーバーを接続する理由
</div>

Lovable では、MCP サーバーを接続することで次の 2 つの重要な機能が有効になります。

- **既存のコンテキストを Lovable に取り込む。**\
  Lovable Agent は、チームのドキュメント、チケット、デザインファイルを読み取り、チームの基準に沿ったプロトタイプやフローを構築できます。
- **Lovable アプリをあなたのワークフローに接続する。**\
  Lovable Agent は、接続済みツール上で後続の処理を実行できます。たとえば、チケットのステータス更新、プロトタイプのリンクをコメントとして追加すること、サポートされている場合は新しいアイテムを作成することなどです。

Lovable があなたのチームの世界を理解すると、手探りの作業がなくなり、反復が高速になり、**アイデアから本番アプリまでこれまで以上の速さで到達**できるようになります。

<div id="how-to-connect-a-prebuilt-mcp-server">
  ### あらかじめ用意された MCP サーバーを接続する方法
</div>

人気のサードパーティアプリやサービス向けのあらかじめ用意されたインテグレーションを使うには、次の手順を実行します。

1. **Settings → Integrations → MCP servers** に移動します。
2. **MCP サーバーを選択**します（例: Notion、Linear、Atlassian など）。
3. **Connect** をクリックします。サインインして、Lovable にアカウントへのアクセスを許可します。

これらの手順に加えて、[n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server) 側で次の設定が必要です。

- **Settings → MCP access** に移動し、**Enable MCP access** をオンにします（インスタンスオーナーまたは管理者権限が必要）
- 公開したい各ワークフローについて、ワークフローエディタ内の **Settings** に移動し、**Available in MCP** をオンにします。

<div id="custom-mcp-servers-business-enterprise">
  ## カスタム MCP サーバー（Business & Enterprise）
</div>

柔軟性を最大限に高めるために、**カスタム MCP サーバー** を接続して、社内システムやサードパーティシステムを Lovable と連携できます。プライベート API、社内 CRM、カスタムデータソースなどに最適です。

Lovable と互換性のある高品質なリモート MCP サーバーを厳選した一覧である [remote MCP servers](https://mcpservers.org/remote-mcp-servers) をぜひご覧ください。

<div id="how-to-connect-a-custom-mcp-server">
  ### カスタム MCP サーバーを接続する方法
</div>

カスタムまたは社内の MCP サーバーを接続するには、次の手順を実行します。

1. **Settings → Integrations → Your MCP servers** に移動します。
2. **New MCP server** の横にある **Set up** をクリックします。
3. **サーバーの詳細を入力します:**
   - **Server name**: サーバーを識別しやすい名前（例: _Internal CRM_ や _Analytics API_）。
   - **Server URL**: Lovable から MCP サーバーにアクセスできるアドレス。
4. **認証:**
   - **OAuth (default):** Lovable はデフォルトで OAuth を使用します。**Add server** をクリックすると、認可が求められます。
   - **Bearer token or API key:** MCP サーバーが OAuth をサポートしていない場合は、**Advanced settings** を展開し、**bearer token または APIキー** を入力します。入力すると、Lovable は OAuth の代わりにこれを使ってリクエストを認証します。
4. **Add server** をクリックします。\
   これでカスタム MCP サーバーが一覧に表示され、ビルド時にコンテキストデータを提供できるようになります。

<div id="security-and-permissions">
  ## セキュリティと権限
</div>

MCP サーバー接続は**ユーザー単位かつ個人用**です。いつでも **Settings → Integrations** から接続を確認したり取り消したりできます。

プロジェクトレベルで素早くアクセスするには、プロンプトボックスの `+` をクリックし、**Integrations** に進みます。

  src="/images/mcp_integrations_project_access.png"

**Business** と **Enterprise** プランでは、ワークスペースの**管理者とオーナー**が、すべてのユーザーに対してどのサーバーを利用可能にするかを管理できます。

- ワークスペース全体の MCP サーバーアクセスを有効または無効にするには、**Settings → Privacy & Security** に移動します。
- 個々の MCP サーバーを有効または無効にするには、**Settings → Integrations → Manage servers** に移動します。

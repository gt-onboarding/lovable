アプリを公開すると、Lovable のプロジェクトが他の人と共有できる本番用のウェブアプリになります。Lovable では、アプリをウェブ上に公開するのが簡単です。一般公開する場合でも、ワークスペース内のメンバーだけに共有を限定する場合でも対応できます。

* **Free および Pro プラン:** 公開したウェブサイトは **公開** 状態となり、リンクを知っている誰でもアクセスできます。これらのプランではアクセス制限ができないため、公開する前に共有の準備ができているか必ず確認してください。
* **Business および Enterprise プラン:** **アプリを閲覧できる人を管理** できます。リンクを知っている全員か、認証済みのワークスペースメンバーのみに限定するかを選択できます。

一度公開した後でも、公開中のアプリはいつでも更新でき、独自ドメインを接続して、より洗練されたブランド体験を提供できます。

<div id="website-access-settings-business-and-enterprise-plans">
  ## ウェブサイトのアクセス設定（Business および Enterprise プラン）
</div>

Business および Enterprise プランでは、公開済みアプリを誰が閲覧できるかを管理し、プライバシーとコントロールを維持できます。

これにより、次のことが可能になります。

* ワークスペース内だけに非公開で保持される**社内向けアプリ**を構築して共有できる。
* チームメイトが誤ってアプリを外部に公開してしまうことを防げる。
* 機密データを扱う組織のガバナンスとコンプライアンスをサポートできる。

アクセスは、**ワークスペースレベル**と**プロジェクトレベル**の両方で管理できます。

<div id="workspace-level-setting">
  ### ワークスペースレベルの設定
</div>

Workspace の**管理者**と**オーナー**は、すべての公開済みアプリに対する既定のアクセスポリシーを設定できます。

**Workspace settings → Privacy &amp; Security → Default website access** に移動し、次のいずれかを選択します。

* **Anyone with link:** すべての公開済みアプリは既定で公開され、誰でもアクセスできます。
* **Workspace members**: すべての公開済みアプリは既定で非公開となり、認証済みの Workspace メンバーのみがアクセスできます。これにより、新しいアプリは明示的に公開しない限り非公開のままになります。

<div id="project-level-setting">
  ### プロジェクトレベルの設定
</div>

個々のプロジェクトごとに、ワークスペースのデフォルト設定を上書きできます。

**Publish** に移動し、**Who can access?** を次のいずれかに設定します:

* **Anyone with link:** URL を知っている人なら誰でもアクセスできます。
* **Workspace members:** サインインしているワークスペースメンバーのみに制限されます。

これを変更しない場合、このプロジェクトはワークスペースのデフォルト設定を継承します。

<div id="publishing-your-project">
  ## プロジェクトを公開する
</div>

<Note>
  Business プランおよび Enterprise プランでは、プロジェクトを公開できるのは
  ワークスペースのオーナーと管理者のみです。
</Note>

1. プロジェクトの画面右上にある **Publish** アイコンをクリックします。
2. **Publish** モーダルで、次の内容を確認・設定します：
   * **Website address**：デフォルトでは、アプリは `[project-url-slug].lovable.app` で公開されます。`[project-url-slug]` を編集するか、有料プランで利用できる [独自ドメイン](/ja/features/custom-domain) を追加できます。
   * **Website info**：アプリアイコン、タイトル、説明、共有用画像などのメタデータをカスタマイズします。
   * 公開したアプリに **アクセスできるユーザー** を選択します（Business プランおよび Enterprise プランでのみ設定可能）：
     * **Anyone with link**
     * **Workspace members**
3. 本番公開前に潜在的なセキュリティ脆弱性を洗い出すため、**Review security** をクリックします。

<Danger>
  公開前には必ずセキュリティレビューを実施してください。これにより、
  機密データの流出や不正アクセスを招く可能性のある脆弱性を特定できます。
  詳細は [Security](/ja/features/security) を参照してください。
</Danger>

4. 準備ができたら **Publish** をクリックします。\
   デプロイが完了すると、公開されたリンクがポップアップで表示されます。

アプリの改善は引き続き行えます。更新を反映したいときは、**Publish** モーダルで **Update** をクリックしてください。

<div id="faq">
  ## FAQ
</div>

<AccordionGroup>
  <Accordion title="サイトを公開できないのはなぜですか？">
    サイトを公開しようとしたときにエラーが表示される場合、ほとんどの場合は、現在のバージョンのサイトが正しく動作していない原因となっている、表面化していないビルドエラーによるものです。Lovable Agent に調査を依頼して、原因の切り分けを試してみてください。
  </Accordion>

  <Accordion title="公開済みサイトに変更内容が反映されないのはなぜですか？">
    変更内容は自動的に本番アプリに反映されません。更新内容を公開するには、**Publish** をクリックし、その後 **Update** をクリックしてください。
  </Accordion>

  <Accordion title="公開済みリンク（ウェブサイトの URL）を変更できますか？">
    はい、できます。プロジェクトの URL スラッグを変更すると、[lovable.app](http://lovable.app) 上のウェブサイトアドレスを、次の 2 通りの方法で変更できます。

    1. **Publish** をクリックし、ウェブサイトアドレスを編集してから、**Update** をクリックします。
    2. **Project settings → Edit URL Slug** に移動し、URL スラッグを変更してから、**Update URL Slug** をクリックします。 

    なお、プロジェクト名を変更してもプロジェクト URL は変わらず、変更されるのはプロジェクトの表示名だけです。
  </Accordion>

  <Accordion title="プロジェクトの公開を解除できますか？">
    はい、できます。プロジェクトの公開は次の 2 通りの方法で解除できます。

    1. **Project settings** に移動し、**Unpublish project** をクリックします。
    2. **Publish** をクリックし、**Advanced** セクションを展開してから **Unpublish** をクリックします。

    公開を解除すると、アプリには誰もアクセスできなくなります。
  </Accordion>
</AccordionGroup>

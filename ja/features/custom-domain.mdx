**独自ドメイン**を使うと、公開済みの Lovable プロジェクトに自分が所有するドメインまたはサブドメインを利用できます。独自ドメインを接続すると、訪問者はデフォルトの `xxx.lovable.app` のプロジェクト URL ではなく、ブランドに合わせた URL を使ってアプリにアクセスできるようになります。

独自ドメインを接続すると、サイトの次の点が向上します：

* **ブランドの一貫性** - より信頼感があり、プロフェッショナルに見える
* **検索エンジンでの露出** - SEO と発見されやすさが向上する
* **記憶しやすさ** - 共有や覚えてもらうことが容易になる

<div id="prerequisites">
  ## 前提条件
</div>

独自ドメインを接続するには、次の条件を満たしている必要があります：

* Lovable の有料プランを利用していること
* プロジェクトが公開されていること。[アプリを公開する](/ja/features/publish)を参照してください。
* ドメイン名を所有していること。**Lovable 上から直接ドメインを購入することもできます**（IONOS 経由）。
* 接続したいドメインまたはサブドメインの **DNS 設定にアクセスできること**。**ドメイン所有権の確認** には、セットアップ時に Lovable から提供される `A` レコードと `TXT` レコードを DNS に追加する必要があります。

<div id="connect-a-custom-domain">
  ## 独自ドメインを接続する
</div>

公開済みの Lovable プロジェクトに、**自動**セットアップ（Entri を利用）または **手動**での DNS 設定を使ってドメインを接続できます。どちらの方法でも、ドメインの所有権を確認する必要があります。

ドメインは次の 2 つの場所から接続できます:

* **Project → Settings → Domains**
* **Publish モーダル → Custom domain**（プロジェクトの公開後でも設定できます）

独自ドメインを接続するには、次の手順に従ってください:

1. **Connect domain** をクリックします。
2. ドメインまたはサブドメイン（例: `yourdomain.com` や `blog.yourdomain.com`）を入力し、**Connect domain** をクリックしてから、**Continue** をクリックします。

   <Info>
     ドメインを追加しても、Lovable は `www` サブドメインを自動的には追加しません。両方を利用したい場合は、`www` も別途追加してください。
   </Info>
3. 次の 2 つの方法のいずれかでドメインを接続します:
   * **Entri を使った自動セットアップ**: 一覧からドメインプロバイダーを選択し、ログインして、Entri に DNS レコードの更新を許可します。
   * **手動セットアップ**: DNS プロバイダーが一覧にない場合は、**Select your domain provider** モーダルの一番下までスクロールし、**Go to our manual setup** を選択します。`A` レコードと `TXT` レコードをコピーし、ドメインレジストラに直接入力します。
4. **Done** をクリックします。検証と SSL 証明書の発行が完了するまで待ちます。完了すると、ドメインが **Live** となり、Lovable アプリに独自ドメインでアクセスできるようになります。

<Info>
  DNS の変更が反映されるまでに最大 **72 時間かかる**場合がありますが、ほとんどの更新は数時間以内に反映されます。
</Info>

<div id="domain-statuses">
  ## ドメインのステータス
</div>

ドメインには、検証の進行状況、DNS の伝播状況、SSL 証明書の準備状況、およびユーザーに必要な操作がひと目でわかるステータスが表示されます。

| Status               | Description                                                                                                                                                                                                                                       | User action                                                                                                                                                                                                 |
| :------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Unpublished**      | プロジェクトがまだ公開されていない場合にのみ、`xxx.lovable.app` のプロジェクト URL に対して表示されます。                                                                                                                                          | プロジェクトを公開します。                                                                                                                                                                                 |
| **Ready**            | ドメインは利用可能ですが、プロジェクトが未公開の状態です。再度公開すると、ステータスは自動的に **Live** に変わります。                                                                                                                            | プロジェクトを公開します。                                                                                                                                                                                 |
| **Action required**  | 独自ドメインの設定が完了していません。                                                                                                                                                                                                            | **Complete setup** をクリックして接続を完了します。                                                                                                                                                        |
| **Verifying**        | DNS レコードの伝播を待機しています。                                                                                                                                                                                                              | **Check status** をクリックして最新のステータスを確認し、DNS レコードを検証します。<br /><br />特に TXT レコードの値を含め、`A` レコードと `TXT` レコードが **DNS プロバイダー側で** 正しく追加されていることを確認してください。 |
| **Unable to verify** | 1 時間以内に DNS 検証が完了しませんでした。通常は、DNS レコードが正しくないか、まだ伝播中であることを意味します。<br /><br />**Note**: *DNS の伝播には最大 72 時間かかる場合がありますが、多くの更新は数時間以内に反映されます。* | **Check again** をクリックしてステータスを更新し、DNS レコードを再検証します。<br /><br />特に TXT レコードの値を含め、`A` レコードと `TXT` レコードが **DNS プロバイダー側で** 正しく追加されていることを確認してください。 |
| **Setting up**       | 検証は完了し、SSL 証明書を発行しています。                                                                                                                                                                                                        | 操作は不要です。ドメインは自動的に **Live** または **Failed** に移行します。                                                                                                                              |
| **Failed**           | 所有権は検証されましたが、SSL 証明書をプロビジョニングできませんでした。                                                                                                                                                                          | **Retry** をクリックして再試行します。                                                                                                                                                                     |
| **Live**             | ドメインが稼働中で、プロジェクトを配信しています。                                                                                                                                                                                                | 操作は不要です。                                                                                                                                                                                           |
| **Offline**          | ドメインは以前は **Live** でしたが、DNS レコードが無効になっています。ドメインはもうプロジェクトを指していません。                                                                                                                                | **Recover** をクリックします。表示されている DNS レコードを確認し、ドメインを再接続するために **DNS プロバイダー側で** 更新します。                                                                      |
| **Removed**          | このドメインは、別のプロジェクトに追加されたか、DNS 検証が完了しなかった（DNS レコードが正しく追加されていない、または伝播が完了しなかった）ため削除されました。                                            | **Reconnect** をクリックして、ドメインを再度接続します。                                                                                                                                                   |

ドメインは、ステータスに関係なくいつでも削除できます。

<div id="primary-domain">
  ## プライマリドメイン
</div>

Lovable では、**1 つのドメインをプライマリドメインとして設定**できます。

* ドメインステータスが **Live** の場合にのみ、プライマリとして設定できます。ドメインの横にある三点メニューをクリックし、**Set as primary** をクリックします。
* プロジェクトに最初に追加した独自ドメインは、自動的にプライマリドメインになります（2025年10月29日以降に追加されたドメインが対象）。
* あるドメインがプライマリの場合、他のすべてのドメインはそのドメインにリダイレクトされます。
* 同時にプライマリにできるドメインは 1 つだけです。
* プライマリドメインは解除できます。その場合、接続されているすべてのドメインが同じようにそのプロジェクトを配信します。プライマリドメインの横にある三点メニューをクリックし、**Unset as primary** をクリックします。

<div id="faq">
  ## FAQ
</div>

<AccordionGroup>
  <Accordion title="独自ドメインは接続できていますが、サイトが表示されません。どうすればいいですか？">
    * [dnschecker.org](https://dnschecker.org/all-dns-records-of-domain.php) のようなツールを使って DNS 設定を再確認してください。DNS の変更が反映されるまでに最長 72 時間かかることがあります。正しい DNS 設定は **Project → Settings → Domains** で確認できます。
    * ブラウザのキャッシュをクリアするか、シークレットウィンドウでテストしてみてください。
    * **Project → Settings → Domains** で、一度ドメインを削除してから再度追加してみてください。
  </Accordion>

  <Accordion title="Lovable からドメインを購入できますか？">
    はい。Lovable 内からパートナーの IONOS を通じて独自ドメインを購入できます。**Project → Settings → Domains** に移動し、**Buy new domain** をクリックして IONOS 経由でドメインを購入してください。これは有料プランでのみ利用可能です。
  </Accordion>

  <Accordion title="独自ドメインを接続すると SEO に影響しますか？">
    はい、独自ドメインを接続すると SEO に良い影響があります。独自ドメインは検索エンジンでのサイトの見つけやすさを向上させ、ユーザーからの信頼性も高めます。

    詳細は、[Implement SEO and GEO best practices](https://docs.lovable.dev/tips-tricks/seo-geo) を参照してください。
  </Accordion>

  <Accordion title="Lovable は独自ドメイン用の SSL 証明書を提供しますか？">
    はい。Lovable アプリに独自ドメインを接続すると、そのドメイン用の SSL 証明書を自動的に生成・インストールします。これにより、サイトは HTTPS 経由で配信され、訪問者のデータが保護されるとともに、サイトの SEO ランキングも向上します。72 時間経っても証明書が発行されない場合は、[Lovable サポート](https://lovable.dev/support) にお問い合わせください。
  </Accordion>

  <Accordion title="ルートドメインではなくサブドメインを接続できますか？">
    はい。ルートドメインの代わりにサブドメイン（例: `blog.yourdomain.com` や `app.yourdomain.com`）を接続できます。セットアップフローでドメインを入力する際に、使用したいサブドメインを含めて入力してください。
  </Accordion>

  <Accordion title="1 つの Lovable プロジェクトに複数のサブドメインを接続できますか？">
    はい。複数のサブドメインを接続できますが、それぞれを個別に **Project → Settings → Domains** で追加・設定する必要があります。たとえば、`blog.yourdomain.com`、`shop.yourdomain.com`、`app.yourdomain.com` を接続できます。
  </Accordion>

  <Accordion title="接続済みドメインはどこで確認できますか？">
    接続済みドメインは次の 2 つの場所で確認できます。

    * **Project → Settings → Domains** - ステータスに関係なく、すべてのドメインが表示されます
    * **Publish モーダル** - **Live** 状態のドメインがすべて表示されます
  </Accordion>

  <Accordion title="別のプロジェクトで使っているドメインを接続できますか？">
    はい。所有権を確認できる限り、別のプロジェクトで使われているドメインやサブドメインを接続できます。プロジェクトが削除されていても問題ありません。
  </Accordion>

  <Accordion title="プライマリドメインはどのように設定しますか？">
    ドメインをプライマリとして設定するには、ステータスが **Live** である必要があります。

    **Project → Settings → Domains** に移動し、**Live** ドメインの横にある 3 点メニューをクリックして、**Set as primary** をクリックします。
  </Accordion>

  <Accordion title="プライマリドメインの設定を解除するにはどうすればよいですか？">
    **Project → Settings → Domains** に移動し、プライマリドメインの横にある 3 点メニューをクリックし、**Unset as primary** をクリックします。
  </Accordion>

  <Accordion title="独自ドメインを削除するにはどうすればよいですか？">
    ステータスに関係なく、いつでもドメインを削除できます。

    1. **Project → Settings → Domains** に移動します。
    2. 該当ドメインの横にある 3 点メニューをクリックし、**Remove** をクリックしてから、確認のためにもう一度 **Remove** をクリックします。

    これで、そのドメインはプロジェクトから切断されます。接続を完全に解除するには、ドメインプロバイダ側で DNS 設定も必ず削除してください。
  </Accordion>

  <Accordion title="xxx.lovable.app のプロジェクト URL を削除できますか？">
    現時点では、プロジェクトから `xxx.lovable.app` のプロジェクト URL を削除する方法はありません。ただし、独自ドメインを追加してプライマリドメインとして設定できます。これにより、訪問者はデフォルトの `xxx.lovable.app` プロジェクト URL ではなく、ブランドされた URL からアプリにアクセスできるようになります。
  </Accordion>

  <Accordion title="www サブドメインは別途接続する必要がありますか？">
    はい、別途追加する必要があり、別のエントリとして表示されます。ドメインをセットアップしても、**www** サブドメインは自動的には含まれません。
  </Accordion>
</AccordionGroup>

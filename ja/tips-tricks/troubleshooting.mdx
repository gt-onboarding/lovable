<div id="build-errors">
  ## ビルドエラー
</div>

Lovable でビルドすると、コーディングがより速く直感的になりますが、どんな優れた AI 開発ツールでも、ときどきつまずくことがあります。コードが想定どおりに動作しなかったり、AI が意図を誤って解釈して思わぬ挙動をすることもあります。

ここでは、問題を解決して、開発を軌道に戻すためのステップバイステップガイドを紹介します。

<iframe width="100%" height="315" src="https://www.youtube.com/embed/0W8bsbHh2p8?si=p4kWhkdu0PEbgGXm" title="YouTube 動画プレーヤー" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerpolicy="strict-origin-when-cross-origin" allowfullscreen />

<Steps>
  <Step title="まず Try to Fix ボタンを使う">
    エラーが表示されたら **Try to Fix** ボタンをクリックします。Lovable がログをスキャンして問題を検出し、すばやく修正を試みます。これが最初に行うべき、いちばん速い対応です。それでもうまくいかない場合は、さらに詳しく原因を調べましょう。
  </Step>
  <Step title="想定外の挙動を診断する">
    コードはエラーなく実行されているのに、意図したとおりに動かないことがあります。これが **想定外の挙動** で、発見もしづらく、修正もしづらい問題です。次を試してみてください。

    - **Revert と再プロンプト**: 問題が発生する直前の状態までロールバックし、より明確に書き直したプロンプトを試します。
  </Step>
  <Step title="より効果的なプロンプトを書く">
    明確で構造化されたプロンプトは、よりよい結果につながります。次の形式を使ってみてください。

    1. **Chat Mode を使い**、「この機能の手順や計画を整理してもらえますか？」と聞きます。
    2. Lovable が提案した **計画を確認し**、必要に応じて修正や説明を依頼します。
    3. **Lovable にプロンプトを送る**: 「では、これをステップごとに実装していきましょう。」
  </Step>
  <Step title="論理的な順序でビルドする">
    構造が重要です。次の流れで進めることをおすすめします。

    1. レイアウトとページを作成する。
    2. Lovable Cloud か外部の Supabase に接続する。
    3. 認証とユーザーロールを設定する。
    4. 機能ロジックを計画・整理する。
    5. Lovable にプロンプトを送り、機能を実装させる。
  </Step>
</Steps>

<div id="general-issues">
  ### 一般的な問題
</div>

<AccordionGroup>
  <Accordion title="すぐに助けが必要なとき">
    Chat-Only Mode に切り替えて、次のように入力してください：「この関数が動作していません。今どういう状況で、これまでに試したことを順番に説明してもらえますか？」
  </Accordion>
  <Accordion title="UI またはレイアウトの問題">
    - 問題が起きている UI のスクリーンショットを AI に直接渡して、より正確に状況を把握できるようにしましょう。
    - Edit ツールを使って、変更したい特定の UI 要素を選択してハイライトし、どう変更したいかを説明します。
    - 例：「ハイライトされているこの要素が見えますか？これを縦方向に中央揃えにして、パディングを増やしてください。」
  </Accordion>
  <Accordion title="完全に行き詰まったとき">
    - プロンプト：「一度視点を引いて考え直してください。エラーを分析して、別のアプローチを提案してください。」
    - タスクをより小さなパートに分割します。
    - エラーが解消しない場合は、revert ボタンを使って元に戻します。
  </Accordion>
  <Accordion title="プレビューが見つからない、またはサンドボックスの起動中から進まない場合">
    プレビューが見つからない、またはサンドボックスが起動中のまま進まない問題は、サンドボックス環境か接続のどちらかに原因がある可能性があります。サンドボックスの問題は一時的なことが多いので、まずはハードリフレッシュを試してください。これが続く場合は、GitHub との同期が壊れているか、接続に問題がある可能性があります。問題が解消しない場合は、[GitHub連携ガイド](https://docs.lovable.dev/integrations/github) を確認してください。
  </Accordion>
</AccordionGroup>

<div id="core-functionalities">
  ### コア機能
</div>

<AccordionGroup>
  <Accordion title="プレビュー画面が真っ白になる">
    - 多くの場合、vite.config.ts ファイルの問題、またはセキュリティヘッダーがプロジェクトに追加されていることが原因です。
    - コード内のビルドエラーまたは構文エラー
    - クラッシュを引き起こす JavaScript の実行時エラー
    - 依存関係の不足や壊れたコンポーネント
    - ページをブロックしている認証またはルーティングの問題

    トラブルシューティング方法: Lovable に vite.config.ts ファイルの確認やセキュリティヘッダーの削除を依頼してください。ブラウザのコンソールでエラーを確認し、以前のバージョンに戻すか、Lovable に空白画面の問題の調査を依頼してください。
  </Accordion>
  <Accordion title="プロジェクトの公開停止または削除">
    Project Settings（プロジェクト設定）でプロジェクトを公開停止（Unpublish）するか削除できます。
  </Accordion>
  <Accordion title="Edge Functions のエラー">
    Cloud タブ -\> ログ で特定のエラーメッセージを確認してください。必要なすべてのシークレットと環境変数が正しく設定されていることを確認してください。Lovable へのプロンプトでエッジ関数のコードに小さな変更を加え、再デプロイを試してください。データベース接続が問題なく動作していることを確認してください。特定のエラーについて Lovable に調査を依頼し、ログで見つかった問題を修正してもらってください。
  </Accordion>
  <Accordion title="プロジェクトが見つからない">
    これは通常、プロジェクトが削除されたことを意味します。意図的に削除していない場合は、プロジェクトの復元についてはサポートまでお問い合わせください。
  </Accordion>
</AccordionGroup>

<div id="need-more-help">
  ### さらにサポートが必要ですか？
</div>

まだ解決しない場合は、次を試してください：

1. 手順を追ってサポートが必要な場合は、Chat mode を使いましょう。
2. UI や機能に関する問題の場合は、以前の安定した状態に戻してください。
3. 有料プランをご利用で、プラットフォーム全体に影響する問題を報告したい場合は、[サポート](https://lovable.dev/support)に連絡してください。

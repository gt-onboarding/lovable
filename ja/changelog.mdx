<Update label="2025年11月26日">
  **MCPサーバー**

  ![Changelog Connectors 11 18 25 Pn](/images/changelog-connectors-11-18-25.png)

  Lovableを業務ツールに接続しましょう：Atlassian（Jira、Confluence）、Notion、Linear、n8n。PRD、チケット、ワイヤーフレームを直接ビルドに取り込むことができます。n8nワークフローを使えば、Salesforce、Slack、Google Sheetsなど400以上のアプリと連携可能です。BusinessプランとEnterpriseプランでは、カスタムMCPの追加や利用可能なサーバーの制御ができます。

  [詳細はこちら](https://docs.lovable.dev/integrations/mcp-servers)

  **デザインビュー**

  ![Changelog Design View 11 26 25 Pn](/images/changelog-design-view-11-26-25.png)

  新しいデザインビューでは、プロジェクトのデザインツール間をスムーズに切り替えられる専用パネルが利用できます。ビジュアル編集の更新に加え、テーマやAI画像生成などの新機能も含まれています。

  [詳細はこちら](https://docs.lovable.dev/features/design)

  **テーマ**

  ![Changelog Themes 11 18 25 Pn](/images/changelog-themes-11-18-25.png)

  ブランド基準を一度設定：カラー、タイポグラフィ、間隔。任意のプロジェクトに適用できます。適用前にテーマをプレビューして、表示を確認できます。

  **すべての新しいビジュアル編集**

  ![Changelog Visual Edits 18 11 25 Pn](/images/changelog-visual-edits-11-18-25.png)

  デザインビューからビジュアル編集にアクセスするか、プロンプトボックスのビジュアル編集ショートカット（旧称：編集）を使用して、左側のパネルで直接編集を行えます。 テキストとアイコンの両方を含むボタンなど、複合コンテンツを持つ要素内のテキストも編集できるようになりました。 複数の要素を同時に選択して編集できます。 ページ上で直接テキストを変更できます。 マージン、パディング、ボーダー、シャドウ、色、アイコンを調整できます。 画像を更新したり、AIで画像を生成できます。

  **ビジュアル編集でのAI画像生成**

  ![Changelog AI Image Gen 18 11 25 Pn](/images/changelog-AI-image-gen-11-18-25.png)

  欲しい画像を説明するだけでカスタム画像を生成できます。Lovableを離れることなく、プロジェクト用のオリジナルビジュアルを作成できます。AI画像生成にLovableクレジットは消費されません。

  **すべてのユーザーがプライベートプロジェクトを利用可能**

  すべてのプランでワークスペース限定プロジェクトが利用可能になりました（デフォルトで有効）。デフォルト設定は「設定」→「プライバシーとセキュリティ」から変更できます。

  [詳細はこちら](https://docs.lovable.dev/features/private-projects)

  **Shopify連携の改善**

  既存の**Shopifyストアを接続**できるようになりました。セキュリティ上の理由により、LovableユーザーのメールアドレスとShopifyストアオーナーのメールアドレスが一致している必要があります。

  **新しいストア**を作成する際、セットアップ中にストアの所在地を選択できるようになり、申請フローがより明確な状態表示で改善されました。ストアが申請されると、申請したLovableユーザーがShopifyストアのオーナーになります。

  認可の更新

  * 既存のストアを接続したユーザー、または新しいストアを申請して取得したユーザー本人のみが、書き込み権限（商品、バリアント、割引コードの作成・更新・削除）を持ちます。
  * 共同編集者は Shopify データ（商品、商品バリエーション、割引コードの検索および取得）には読み取り専用でしかアクセスできませんが、ストアフロントは引き続き制限なく構築できます。

  **Gemini 3 Pro &amp; 画像生成に対応**

  Lovable AIは、Gemini 3 ProやNano Banana Proを含むGoogleの最新モデルをサポートするようになりました。Lovable Cloud &amp; AIを有効化して、アプリでこれらのモデルを利用できます。

  **共有・招待機能の改善**

  共有ダイアログが再設計され、プロジェクト内で誰が何をできるかがより明確になりました。権限の可視性の向上、共有プレビューの強調表示、コラボレーション概念の明確化を実現しています。

  **クラウドストレージの改善**

  Cloudビューから直接ストレージを管理します。プライベートまたは公開バケットを作成し、フォルダを整理して、ファイルのアップロード、名前変更、削除、ダウンロードを行えます。プライベートバケットのファイルもUIに表示されるようになりました。

  **質問**

  リクエストに複数の実装方法がある場合、Lovableは構築前に確認のための質問をするようになりました。複数の選択肢から選ぶか、独自の入力を追加することで、Lovableがあなたの求めている内容を正確に理解できるようになります。

  **統合機能**

  プロジェクト設定の新しい「統合」タブでは、Lovable Cloud、Lovable AI、MCPサーバー、Supabase、Shopify、Stripeを一元管理できます。

  **チャット提案**

  コンテキストに応じたプロンプトがチャットに表示され、次に構築すべき内容を提案します。

  **クレジット残高不足の通知**

  ProおよびBusinessワークスペースのユーザーには、ビルドクレジットの残数が5未満になるとアラートが表示され、月間クレジットを増やすためのプランアップグレードへのショートカットも併せて表示されます。このアラートはプロンプトボックスの上部、プロジェクトメニュー、ワークスペースメニューに表示され、ビルド中にクレジットが不足する事態を防ぎます。

  **教育用の一時ログイン**

  教育者は、授業で使用する一時的な学生アカウントを lovable.dev/login/temp で作成できます。

  **リミックス機能の改善**

  プロジェクトのリミックス時に、進行状況をステップごとに確認できます。リミックスできない場合は、その理由を説明する明確なエラーメッセージが表示されます。リミックスはバックグラウンドで実行され、通知が継続表示されるため、他の作業を並行して進められます。カスタムシークレットを使用したプロジェクトのリミックスにも対応しました。

  **改善された復元機能とメッセージの編集・復元機能**

  「復元」が「元に戻す」に変更されました。チャット履歴の任意の時点に戻ったり、過去のメッセージを編集して新しい方向性を試すことができます。元の作業内容はチャットに保持されており、いつでも再適用可能です。

  **プロジェクトURL**

  プロジェクト作成時にプロジェクトURLが自動生成されなくなりました。公開前に独自のプロジェクトURLを設定するか、Lovableに自動生成させてください。

  **プロジェクト表示名**

  Lovableは、プロジェクトに対してより分かりやすい表示名を自動生成するようになりました。表示名は、あなたとワークスペースのメンバーにのみ表示され、公開したアプリの訪問者には表示されません。表示名はワークスペース内でのみ一意であるため、既に使用されている名前との競合が軽減されます。プロジェクト設定から、いつでもプロジェクト名をより分かりやすいものに変更できます。

  **より良いコミットメッセージ**

  コミットメッセージに、依頼内容だけでなく、プロジェクトで実際に変更された内容が記載されるようになり、プロジェクト履歴の確認が簡単になりました。

  **ナビゲーションの更新**

  プライバシーとセキュリティの設定が専用タブに移動し、アクセスしやすくなりました。

  **バグ修正:**

  * ページの再読み込みを行うまで「Try to Fix」のまま進まなくなっていたプレビューのリロードエラーを修正しました。システムが一時的なエラーを検知し、自動的にリトライするようになりました。
  * 品質上の問題により、Figma インポート機能は廃止されました。現在、新しくてエキサイティングな機能を開発中ですので、続報をお待ちください。
</Update>

<Update label="2025年11月5日">
  **Lovable × Shopify 連携**

  ![image](/images/changelog_shopifyxlovable.png)

  AI とチャットするだけでオンラインストアを構築できます。Lovable が商品、カート、チェックアウトをセットアップし、準備ができたら Shopify 上でストアを引き継いで公開できます。趣味のショップから大規模なカタログまで幅広く活用できます。

  実際の動作は、Lovable のグッズストアで確認できます: lovable.dev/merch

  おまけ: Lovable 経由で登録すると、Shopify の 30 日間トライアルが利用できます。

  **エージェントに「何をしたいか」を伝えやすく**

  ![image](/images/changelog_screenshot_capture.png)

  Lovable エージェントが、あなたのコンテキストをこれまで以上によく理解できるようになりました。いま Lovable のどのタブ（Code Editor、SEO、Analytics、Cloud 設定、Secrets など）を見ているかを把握し、より関連性の高い回答を返せます。エージェントにビルドのスクリーンショットを撮らせたり、チャット画面からコンピューター上のどこからでもスクリーンショットをキャプチャしたりできます。

  **Cloud & AI の残高を常に十分に確保**\
  残高が \$10 未満になると自動でチャージされますが、月額上限を設定して請求額はしっかりコントロールできます。各プロジェクトごとの Cloud & AI の使用状況も確認できるようになりました。

  **独自ドメイン管理の改善**\
  プロジェクトのプライマリドメインを設定でき、他の独自ドメインは自動的にそこへリダイレクトされます。新しいドメインの追加も、よりわかりやすいフローとステップごとのガイドにより簡単になりました。さらに、ドメインを公開する前に所有権を確認するためのドメイン検証機能も追加しました。

  **非公開プロジェクトへのアクセスをリクエスト**\
  非公開プロジェクトを開こうとしたとき、プロジェクトオーナーにアクセスをリクエストできるようになりました。非公開の作業を安全に保ちつつ、コラボレーションをしやすくします。

  **ワークスペース限定公開（Business & Enterprise プラン）**\
  公開済みプロジェクトを「workspace-only」アクセスに設定できるようになり、認証を必須にして、あなたのワークスペースメンバーのみにアクセスを制限できます。社内向けツールやアプリを構築するのに最適です。

  **デスクトップ通知**\
  実行に時間がかかるビルドが完了したタイミングを知るために、デスクトップ通知を有効にできます。終わったかどうかを何度も確認しに戻る必要はありません。

  **プロジェクトを GitHub から切断**\
  必要に応じて、プロジェクトを GitHub から切断できるようになりました。別の GitHub 組織にプロジェクトを移管する必要がある場合などに便利です。

  **バグ修正**

  - エージェントの回答が、タイピング表示ではなく一度にまとめて表示されてしまう問題を修正しました。
  - サインアップ直後に表示されるポップアップ通知の数を減らしました。
  - 問題が発生した際に、エラーメッセージがより一貫して表示されるようになりました。
  - チャット画面内の余白や配置の問題を修正しました。
</Update>

<Update label="Oct 6, 2025">
  **Lovable Cloud**

  ![image](/images/lvbl-cloud-logo.png)

  Supabase を手動でセットアップすることなく、フルスタックアプリを構築できます。Lovable Cloud なら、オンデマンドのデータベース、ユーザー認証、ストレージを提供し、アプリの成長に合わせて自動的にスケールします。

  Lovable Cloud は、すべての新規プロジェクトと、Supabase と統合されていない既存のプロジェクトで利用できます。詳しくは[こちら](https://docs.lovable.dev/features/cloud)をご覧ください。

  **Lovable AI**

  ![image](/images/lvbl-ai-logo.png)

  今では、Lovable でプロンプトを入力するだけで、AI ネイティブなアプリを構築できます。プロバイダーへのアクセスや API の管理は私たちが行うので、あなたはプロダクト作りに集中できます。10 月 13 日まで、Google Gemini 搭載の埋め込み AI 機能を無料でご利用いただけます。

  例えば、次のようなものを作成できます:

  - **特化型 ChatGPT クローン** — サポート、セールス、ナレッジワーク向けに調整された会話型アシスタント。
  - **アバター & 画像ジェネレーター** — ユーザーがビジュアルコンテンツを作成・編集・カスタマイズできるようにします。
  - **コーチングアプリ** — パーソナライズされたガイダンス、オンボーディングコーチ、習慣化サポーターなど。

  すべてのプランに、埋め込み AI 機能に利用できる毎月の無料クレジットが含まれます。無料プランも対象です。完全な料金表は[こちら](https://docs.lovable.dev/features/cloud#usage-based-cloud-and-ai-pricing)から確認できます。

  **Voice Mode**

  Lovable と会話できるようになりました。アイデアや編集内容を声で説明すると、Lovable がそれを形にしてくれます。より直感的で会話的なビルド体験が可能になります。

  **ファイルをそのままアプリに変換**

  ![image](https://lovable.dev/_next/image?url=%2Fcontent%2Fnews%2Fcovers%2Fiboxasset.png&w=1920&q=75)

  ほぼあらゆる種類のファイルを Lovable にドラッグ＆ドロップして、アプリに変換できます。

  例えば:

  - シート → インタラクティブなダッシュボード
  - 履歴書 → ポートフォリオ
  - スライド資料 → カスタムプレゼンテーション

  **Sonnet 4.5**

  Anthropic の最新モデルが、Lovable 上でのビルドをさらに強化します。より強力な推論、より一貫した結果、マルチステップの編集や構造化データ処理におけるパフォーマンス向上が期待できます。

  **よりスムーズなビルド体験**

  Lovable でのビルド体験がさらにスムーズになりました。Visual Editor へのより素早いアクセス、長時間利用時の UX 改善、Code Editor 内での画像サポートなどが含まれます。

  **安心して公開・非公開を切り替え**

  安心してリリースできます。公開フローに、ライブ公開前のリンクプレビューの編集機能が追加されました。間違えても大丈夫です。ワンクリックで非公開に戻せます。

  **Stripe 連携**

  Stripe 連携を改善し、課金・支払いフローがより分かりやすくなりました。作りたい内容をチャットで説明するだけで、すべての Stripe ロジックをセットアップできます — 手動設定は不要です。

  **コミュニティリソースのアップデート**

  Partner ページとリミックスの両方がアップグレードされ、コミュニティを通じてあなたのビルドをさらに拡張できるようになりました。Partner ページでは専門エージェンシーを探したり（または掲載申請をしたり）でき、好きな Lovable Cloud プロジェクトでリミックスを使うことで、より素早くスタートできます。

  **より良いワークスペース管理**

  メンバーを一元的に確認できる新しいビューにより、チーム全体の利用状況を把握し、必要なアクションを実行できるようになりました。管理者は、セッションの有効期間制限の設定、ジャストインタイムプロビジョニング（SSO）の有効化、公開の無効化、プロジェクトのプライバシー設定の強制が可能です。

  **バグ修正**

  - 横スクロールを引き起こしていたチャット内のサイズ問題を修正
  - ビルドエラーを 20% 削減しました
</Update>

<Update label="Sep 1, 2025">
  **学生・教職員向け割引が利用可能に**

  学生と教育関係者の方が、より簡単に開発を始められるようになりました。今なら、学生・教職員の方は Pro を最大 1 年間、最大 50% オフでご利用いただけます。無料ワークスペースから始める場合に割引が適用されるので、予算を抑えながら Lovable のフル機能を利用できます。[このオファーの詳細はこちら](https://lovable.dev/students)。

  **セキュリティの強化**

  ![image](https://userimg-assets-eu.customeriomail.com/images/client-env-165734/1756734639962_security_01K42RPM40HYH5PH5QS40Z6FMY.png)

  セキュリティは私たちの最優先事項です。SOC 2 Type 2 準拠や ISO 27001:2022 認証の取得を含め、セキュリティ機能・ポリシー・認証を拡充しました。また、優れた CISO の Igor Andriushchenko を迎えました。あなたのアイデアやデータ、そしてお客様を安心して預けていただけます。セキュリティ機能の詳細は[こちら](https://lovable.dev/security)をご覧ください。

  **新しい Lovable Inbox**

  ![image](https://userimg-assets-eu.customeriomail.com/images/client-env-165734/1756734786651_inbox_01K42RV3BXGH7MP6MYJ1KHKQH8.png)

  新しいパーソナル Inbox では、プロジェクトやワークスペースへの招待を追跡でき、最近リリースされたアップデートも確認できます。常に最新の状況を把握できます。

  **新しいアプリ向けの、よりスマートな SEO 対応** 新しいアプリやウェブページを作成すると、Lovable が SEO のベストプラクティスをより適切に反映した、エージェント型の SEO 指示を自動で含めるようになりました。これにより、あなたのプロジェクトが検索エンジンにより良くインデックスされ、ターゲットとするユーザーに届きやすくなります。

  **Workspace Viewer ロール（Pro+）** Pro+ メンバー向けに、新しい Viewer ロールを追加しました。Viewer はワークスペースのメタデータ、プロジェクト、メンバーを閲覧できますが、プロジェクトの作成・編集や他ユーザーの招待はできません。編集は不要だが状況を把握したいステークホルダーに最適なロールです。

  **柔軟なプラン管理**

  ![image](https://userimg-assets-eu.customeriomail.com/images/client-env-165734/1756734900964_Screenshot%202025-08-28%20at%205.05.18%20pm_01K42RYK0KYRNYETKVWAW22NFR.png)

  数クリックで、必要に応じてプランのアップグレードやダウングレードができるようになりました。より多くのクレジットが必要なときも、無料プランにスケールダウンしたいときも柔軟に対応できます。

  **エラー時のフィードバックがより明確に**

  ![image](https://userimg-assets-eu.customeriomail.com/images/client-env-165734/1756734470366_image%20\(4\)_01K42RHEPDYDCZT5NG96S3TEGY.png)

  LLM がリクエストを誤って解釈した場合、Lovable が直接お知らせするようになりました。何がうまくいかなかったかが分かるので、プロンプトを調整して、より素早く前に進めます。

  **編集履歴と連続利用状況をトラッキング**

  ![image](https://userimg-assets-eu.customeriomail.com/images/client-env-165734/1756734477303_GyzkRM5bEAAz-_N_01K42RHN8S62MTE2T7KC65JCB3.jpeg)

  Lovable で自分が行った編集回数や、連続利用状況（ストリーク）を確認できる個人用ダッシュボードを追加しました。モチベーションを維持し、時間の経過とともに自分の成長を実感するのに役立ちます。Account Settings から確認してみてください。

<Update label="2025年8月11日">
  **Agent Mode が新しいデフォルトモードになりました**

  このアップグレードにより、Lovable は真のエージェントパートナーになります。リクエストを解釈し、コードベースを理解し、不具合を修正し、ファイルをまたいだ複雑なマルチステップ編集を実行し、外部ツールと連携し、エラーを 91% 削減することで、より野心的なアプリを、これまでより速く、少ない手間で構築できるようになります。

  エージェントを使わない Legacy mode は 2025年9月1日に廃止されます。

  **Lovable アプリのリアルタイム分析**

  公開中のあなたのアプリがどのように使われているか気になりませんか？Lovable の中から直接、ライブトラフィックとエンゲージメントを追跡できるようになりました。たとえば次のような指標です：

  - Visitors: 何人があなたのアプリを使っているか Pageviews: どのページが最も注目されているか
  - Visit duration: ユーザーがどのくらい滞在しているか
  - Bounce rate: どのくらいの割合がすぐに離脱しているか
  - Views per visit: ユーザーがどれだけ深く利用しているか

  データはほぼリアルタイムで更新されるため、トレンドの把握、不具合のデバッグ、あるいは最初のユーザー獲得をお祝いするのにも役立ちます。Project Settings → Analytics から今すぐ利用できます。

  **chat mode の料金変更**

  chat mode の料金体系は、メッセージあたり 1 クレジットから、Agent Mode と同じモデルであるダイナミックプライシングに移行します。これは、各メッセージとレスポンスのコストが、その内容の複雑さに基づいて決まることを意味します。つまり、シンプルなメッセージやレスポンスは 1 クレジット未満になり、非常に複雑なものは 1 クレジットを超える場合があります。この変更は 2025年9月1日から全ユーザーに順次適用されます。

**MCP (Model Context Protocol)** es un estándar abierto que conecta sistemas de IA, como Lovable, con herramientas, servicios y fuentes de datos externas.

Al conectar Lovable con herramientas como Notion, Atlassian o Linear, el Agente de Lovable puede usar el contexto real de tu equipo para generar código, prototipos y documentación más precisos.

Cuando conectas un servidor MCP, Lovable puede leer datos estructurados (como texto de documentos, incidencias o diagramas) y, cuando la integración lo permita, realizar acciones limitadas como crear o actualizar elementos.

Lovable incluye varios servidores MCP preconfigurados en todos los planes, con la opción de añadir servidores personalizados en los planes Business y Enterprise.

<div id="why-connect-mcp-servers">
  ## Por qué conectar servidores MCP
</div>

En Lovable, los servidores MCP habilitan dos capacidades clave:

- **Incorpora tu contexto existente a Lovable.**\
  El agente de Lovable puede leer la documentación de tu equipo, los tickets y los archivos de diseño para crear prototipos y flujos alineados con tus estándares.
- **Conecta tus apps de Lovable con tus flujos de trabajo.**\
  El agente de Lovable puede realizar acciones de seguimiento en tus herramientas conectadas — como actualizar el estado de los tickets, añadir enlaces a prototipos como comentarios o crear nuevos elementos cuando la herramienta lo permita.

Lovable se vuelve más potente cuando entiende el mundo de tu equipo — eliminando la incertidumbre, acelerando la iteración y ayudándote a pasar de **idea a app en producción más rápido que nunca.**

<div id="how-to-connect-a-prebuilt-mcp-server">
  ### Cómo conectar un servidor MCP preconfigurado
</div>

Para usar integraciones preconfiguradas con aplicaciones y servicios de terceros populares:

1. Ve a **Settings → Integrations → MCP servers**.
2. **Selecciona un servidor MCP** (por ejemplo, Notion, Linear o Atlassian).
3. Haz clic en **Connect**. Inicia sesión y concede a Lovable acceso a tu cuenta.

Además de estos pasos, debes hacer lo siguiente en [n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server):

- Ve a **Settings → MCP access** y activa **Enable MCP access** (requiere permisos de propietario de la instancia o de administrador).
- Para cada workflow que quieras exponer, ve a **Settings** en el editor de workflows y activa **Available in MCP.**

<div id="custom-mcp-servers-business-enterprise">
  ## Servidores MCP personalizados (Business & Enterprise)
</div>

Para una flexibilidad total, puedes conectar **servidores MCP personalizados** para integrar sistemas internos o de terceros en Lovable, ideal para APIs privadas, CRM internos o fuentes de datos personalizadas.

Explora los [servidores MCP remotos](https://mcpservers.org/remote-mcp-servers), una lista seleccionada de servidores MCP remotos de alta calidad compatibles con Lovable.

<div id="how-to-connect-a-custom-mcp-server">
  ### Cómo conectar un servidor MCP personalizado
</div>

Para conectar un servidor MCP personalizado o interno:

1. Ve a **Settings → Integrations → Your MCP servers.**
2. Haz clic en **Set up** junto a **New MCP server**.
3. **Introduce los detalles del servidor:**
   - **Server name**: Un nombre claro para tu servidor (por ejemplo, _Internal CRM_ o _Analytics API_).
   - **Server URL**: La dirección a la que Lovable puede acceder para comunicarse con tu servidor MCP.
4. **Autenticación:**
   - **OAuth (default):** Lovable usa OAuth de forma predeterminada. Después de hacer clic en **Add server**, se te pedirá que autorices.
   - **Bearer token or API key:** Si el servidor MCP no admite OAuth, despliega **Advanced settings** e introduce un **bearer token** o una **clave API**. Si lo proporcionas, Lovable lo usará para autenticar las solicitudes en lugar de OAuth.
4. Haz clic en **Add server**.\
   El servidor MCP personalizado ahora aparece en tu lista y puede proporcionar datos contextuales para tus builds.

<div id="practical-examples">
  ## Ejemplos prácticos
</div>

Cada app conectada pasa a estar disponible al instante como contexto para el Lovable Agent. Puedes interactuar con las herramientas conectadas usando instrucciones en lenguaje natural.

| **Escenario**                                         | **Ejemplo de instrucción**                                      | **Resultado de ejemplo**                                                                |
| :---------------------------------------------------- | :--------------------------------------------------------------- | :-------------------------------------------------------------------------------------- |
| Una **product manager** conecta **Linear** o **Jira** | _Genera un prototipo basado en los tickets del sprint actual._  | Lovable lee las descripciones de los issues y los criterios de aceptación para crear una UI alineada y comprobable. |
| Una **especialista en marketing** vincula **Notion** o **Confluence** | _Convierte nuestro brief de la campaña de Q4 en una landing page._ | Lovable usa el documento como contexto para generar una página editable alineada con la marca. |
| Un **ingeniero** vincula **n8n** o **MCP interno**    | _Crea una UI de workflow que liste usuarios y permita eliminarlos._ | Lovable crea una app que utiliza datos en tiempo real y workflows.                      |

<div id="security-and-permissions">
  ## Seguridad y permisos
</div>

Las conexiones del servidor MCP son **por usuario y personales**. Puedes revisar o revocar conexiones en cualquier momento en **Settings → Integrations**.

Para acceder rápidamente a nivel de proyecto, haz clic en `+` en el cuadro de instrucciones y ve a **Integrations**.

En los planes **Business** y **Enterprise**, los **administradores y propietarios** del Workspace pueden gestionar qué servidores están disponibles para todos los usuarios:

- Ve a **Settings → Privacy & Security** para habilitar o deshabilitar el acceso a todos los servidores MCP para todo el Workspace.
- Ve a **Settings → Integrations → Manage servers** para habilitar o deshabilitar servidores MCP individuales.

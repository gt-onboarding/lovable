<div id="build-errors">
  ## Errores de compilación
</div>

Crear con Lovable hace que programar sea más rápido e intuitivo, pero incluso las mejores herramientas de desarrollo con IA pueden encontrarse con algún obstáculo de vez en cuando. A veces tu código no se ejecuta como esperas o puedes notar un comportamiento inesperado donde la IA interpreta mal tu intención.

Aquí tienes una guía paso a paso para ayudarte a resolver problemas y retomar el rumbo:

<iframe width="100%" height="315" src="https://www.youtube.com/embed/0W8bsbHh2p8?si=p4kWhkdu0PEbgGXm" title="Reproductor de video de YouTube" frameborder="0" allow="accelerometer; autoplay; clipboard-write; encrypted-media; gyroscope; picture-in-picture; web-share" referrerpolicy="strict-origin-when-cross-origin" allowfullscreen />

<Steps>
  <Step title="Usa primero el botón Try to Fix">
    Haz clic en el botón **Try to Fix** cuando aparezca un error. Lovable analizará tus registros, detectará el problema e intentará una corrección rápida. Es tu movimiento inicial más rápido. Si eso no funciona, es momento de investigar más a fondo.
  </Step>
  <Step title="Diagnostica el comportamiento inesperado">
    Tu código puede ejecutarse sin errores pero no comportarse como esperabas. Eso es **comportamiento inesperado**, más difícil de detectar y de corregir. Prueba esto:

    - **Revierte y vuelve a formular la instrucción**: Vuelve al punto justo antes de que apareciera el problema y luego prueba con una instrucción más clara y revisada.
  </Step>
  <Step title="Escribe instrucciones más efectivas">
    Instrucciones claras y estructuradas generan mejores resultados. Usa este formato:

    1. **Usa Chat Mode** y pregunta: "¿Puedes describir los pasos o el plan para esta funcionalidad?"
    2. **Revisa el plan** que sugiera Lovable y pide cambios o aclaraciones si es necesario.
    3. **Indícale a Lovable**: "Ahora, construyamos esto paso a paso."
  </Step>
  <Step title="Crea en un orden lógico">
    La estructura importa. Sigue este flujo recomendado:

    1. Crea el diseño y las páginas.
    2. Conéctate a Lovable Cloud o a una conexión externa de Supabase.
    3. Configura la autenticación y los roles de usuario.
    4. Planifica y organiza la lógica de las funcionalidades.
    5. Indícale a Lovable que implemente las funcionalidades.
  </Step>
</Steps>

<div id="general-issues">
  ### Problemas generales
</div>

<AccordionGroup>
  <Accordion title="¿Necesitas ayuda rápida?">
    Entra en el modo solo chat y escribe: "Esta función específica no está funcionando. ¿Puedes explicarme paso a paso qué está pasando y qué has intentado?"
  </Accordion>
  <Accordion title="Problemas de UI o diseño">
    - Proporciona capturas de pantalla del problema de UI directamente a la IA para obtener un diagnóstico más claro.
    - Usa la herramienta Edit para seleccionar y resaltar los elementos específicos de la UI que quieras cambiar y luego describe el ajuste que quieres.
    - Por ejemplo: "¿Ves el elemento resaltado? Centralo verticalmente y aumenta el padding (espaciado interno)."
  </Accordion>
  <Accordion title="Cuando estés completamente atascado">
    - Instrucción: "Da un paso atrás. Analiza el error y sugiere un enfoque diferente."
    - Divide la tarea en partes más pequeñas.
    - Usa el botón de revertir si los errores persisten.
  </Accordion>
  <Accordion title="Vista previa no encontrada o sandbox iniciándose">
    Los problemas con la vista previa no encontrada o con el sandbox iniciándose pueden estar relacionados con el entorno de sandbox o con tu conexión. Los problemas de sandbox suelen ser temporales; prueba primero con una recarga forzada. Si esto continúa, puede indicar una sincronización rota con GitHub o un problema de conexión. Para problemas persistentes, revisa nuestra [guía de integración con GitHub](https://docs.lovable.dev/integrations/github).
  </Accordion>
</AccordionGroup>

<div id="core-functionalities">
  ### Funcionalidades principales
</div>

<AccordionGroup>
  <Accordion title="La vista previa aparece en blanco">
    - A menudo se debe a problemas en el archivo vite.config.ts o a que se han introducido encabezados de seguridad en el proyecto.
    - Errores de compilación o errores de sintaxis en el código
    - Errores de ejecución de JavaScript que provocan bloqueos
    - Dependencias que faltan o componentes dañados
    - Problemas de autenticación o de enrutamiento que bloquean la página

    Para solucionar el problema: pídele a Lovable que revise el archivo vite.config.ts o que elimine los encabezados de seguridad. Revisa la consola del navegador en busca de errores, vuelve a una versión anterior o pídele a Lovable que investigue el problema de la pantalla en blanco.
  </Accordion>
  <Accordion title="Anular la publicación o desactivar proyectos">
    Puedes anular la publicación o eliminar tu proyecto en Project Settings.
  </Accordion>
  <Accordion title="Error con Funciones Edge">
    Revisa la pestaña Cloud -\> Registros para ver mensajes de error específicos. Verifica que todos los secretos y variables de entorno requeridos estén configurados correctamente. Intenta volver a desplegar haciendo un pequeño cambio en el código de la función perimetral pidiéndoselo a Lovable mediante una instrucción. Asegúrate de que tu conexión a la base de datos esté funcionando. Pídele a Lovable que investigue el error específico y solucione cualquier problema que se encuentre en los registros.
  </Accordion>
  <Accordion title="Proyecto no encontrado">
    Normalmente esto significa que el proyecto fue eliminado. Si no lo eliminaste a propósito, comunícate con el equipo de soporte para que te ayuden a restaurar tu proyecto.
  </Accordion>
</AccordionGroup>

<div id="need-more-help">
  ### ¿Necesitas más ayuda?
</div>

¿Sigues atascado? Prueba esto:

1. Usa Chat mode para obtener ayuda paso a paso.
2. Si se trata de un problema con la UI o una función, vuelve a un estado estable.
3. Si eres un cliente de pago y quieres informar de un problema a nivel de plataforma, ponte en contacto con [Soporte](https://lovable.dev/support).

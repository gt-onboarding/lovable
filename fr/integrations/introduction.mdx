Lovable propose des intégrations natives fluides pour exporter du code et ajouter un back-end. Vous pouvez également intégrer une API externe si vous souhaitez ajouter une fonctionnalité précise à votre site web.

## Qu&#39;est-ce qu&#39;une API ?

Une API est une façon pour des applications de communiquer entre elles et de partager des données.
Différentes API proposent différentes fonctionnalités, comme l’envoi d’e‑mails ou le traitement de paiements.
Vous pouvez en savoir plus sur la puissance des API dans [cet article sur notre blog](https://lovable.dev/blog/2025-01-28-the-power-of-apis).

<div id="using-any-api-with-lovable">
  ## Utiliser n’importe quelle API avec Lovable
</div>

Toutes les autres API appelables peuvent toujours être intégrées, mais elles nécessitent souvent un contexte et des instructions supplémentaires.

Vous pourriez avoir :

* **Systèmes internes avec des API personnalisées**\
  Il peut s’agir de services que votre équipe a créés (par ex. systèmes de gestion des stocks, pipelines d’analyse, outils internes) qui exposent des endpoints REST ou GraphQL mais ne sont pas documentés publiquement ni largement connus.
  Vous voulez que Lovable récupère ou envoie des données vers ces systèmes pour automatiser des workflows ou alimenter des interfaces utilisateur.
* **API tierces que Lovable ne prend pas encore en charge**\
  Peut-être que vous utilisez un produit SaaS de niche, un fournisseur de données spécialisé ou une intégration partenaire que Lovable ne connaît pas nativement.
  Même si ce n’est pas officiellement pris en charge, vous voulez quand même que Lovable puisse se connecter, effectuer des appels et travailler avec les données.

Dans les deux cas, le défi est le même : par défaut, Lovable ne « connaît » pas l’API. Mais avec une bonne documentation et une configuration d’authentification correcte, il peut tout de même l’utiliser efficacement.
Dans tous les cas, l’approche est la même : **Lovable peut quand même s’y connecter**. Voici comment.

<Steps>
  <Step title="Donner à Lovable le contexte dont il a besoin">
    Lovable fonctionne au mieux lorsqu’il comprend l’API. Vous pouvez fournir :

    * **Spécification OpenAPI** (si disponible)
    * **Documentation ou exemples copiés-collés** (en particulier les endpoints et les exemples de requêtes/réponses)

    Plus la documentation est claire, mieux Lovable peut déterminer comment appeler l’API.
  </Step>

  <Step title="Gérer l’authentification de manière sécurisée">
    Pour les API qui nécessitent une authentification :

    **Option 1 : Utiliser les Supabase Edge Functions comme proxy**

    * Stocker les Secrets de façon sécurisée sur Supabase
    * Laisser Lovable appeler l’endpoint API via la Fonction Edge (et non l’API directement)

    **Option 2 : Implémenter l’intégration de l’API directement dans votre Projet Lovable**

    * Exposer un endpoint d’authentification (par ex. pour échanger des identifiants contre un jeton ou une session)
    * Laisser Lovable appeler cet endpoint d’authentification dans le cadre du flux d’intégration
    * Inclure les jetons, en-têtes ou informations de session dans les appels API suivants
  </Step>
</Steps>

**MCP (Model Context Protocol)** est un standard ouvert qui connecte des systèmes d’IA, comme Lovable, à des outils, services et sources de données externes.

En connectant Lovable à des outils comme Notion, Atlassian ou Linear, le Lovable Agent peut utiliser le contexte réel de votre équipe pour générer un code, des prototypes et de la documentation plus précis.

Quand vous connectez un serveur MCP, Lovable peut lire des données structurées (par exemple du texte provenant de documents, tickets ou diagrammes) et, lorsque c’est possible, effectuer des actions limitées comme créer ou mettre à jour des éléments.

Lovable inclut plusieurs serveurs MCP intégrés dans toutes les offres, avec la possibilité d’en ajouter des personnalisés dans les offres Business et Enterprise.

<div id="why-connect-mcp-servers">
  ## Pourquoi connecter des serveurs MCP
</div>

Dans Lovable, les serveurs MCP activent deux fonctionnalités clés :

- **Apportez votre contexte existant dans Lovable.**\
  Le Lovable Agent peut lire la documentation de votre équipe, vos tickets et vos fichiers de design pour créer des prototypes et des parcours conformes à vos standards.
- **Connectez vos apps Lovable à vos workflows.**\
  Le Lovable Agent peut effectuer des actions de suivi dans vos outils connectés — comme mettre à jour le statut des tickets, ajouter des liens de prototype en commentaires ou créer de nouveaux éléments lorsque cela est pris en charge.

Lovable devient plus puissant lorsqu’il comprend le monde de votre équipe — éliminant les approximations, accélérant l’itération et vous aidant à passer de **l’idée à l’app en production plus rapidement que jamais.**

<div id="how-to-connect-a-prebuilt-mcp-server">
  ### Comment connecter un serveur MCP préconfiguré
</div>

Pour utiliser des intégrations préconfigurées avec des applications et services tiers populaires :

1. Va dans **Settings → Integrations → MCP servers**.
2. **Sélectionne un serveur MCP** (par exemple Notion, Linear ou Atlassian).
3. Clique sur **Connect**. Connecte-toi et accorde à Lovable l’accès à ton compte.

En plus de ces étapes, tu dois effectuer les actions suivantes dans [n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server) :

- Va dans **Settings → MCP access** et active **Enable MCP access** (nécessite les autorisations de propriétaire ou d’admin de l’instance).
- Pour chaque workflow que tu veux rendre disponible, va dans **Settings** dans l’éditeur de workflow et active **Available in MCP.**

<div id="custom-mcp-servers-business-enterprise">
  ## Serveurs MCP personnalisés (Business & Enterprise)
</div>

Pour une flexibilité maximale, vous pouvez connecter des **serveurs MCP personnalisés** afin d’intégrer des systèmes internes ou tiers dans Lovable — idéal pour des API privées, des CRM internes ou des sources de données personnalisées.

Explorez les [serveurs MCP distants](https://mcpservers.org/remote-mcp-servers), une liste soigneusement sélectionnée de serveurs MCP distants de haute qualité compatibles avec Lovable.

<div id="how-to-connect-a-custom-mcp-server">
  ### Comment connecter un serveur MCP personnalisé
</div>

Pour connecter un serveur MCP personnalisé ou interne :

1. Va dans **Settings → Integrations → Your MCP servers.**
2. Clique sur **Set up** à côté de **New MCP server**.
3. **Renseigne les détails du serveur :**
   - **Server name** : Un nom explicite pour ton serveur (par exemple, _Internal CRM_ ou _Analytics API_).
   - **Server URL** : L’adresse à laquelle Lovable peut accéder à ton serveur MCP.
4. **Authentification :**
   - **OAuth (par défaut) :** Lovable utilise OAuth par défaut. Après avoir cliqué sur **Add server**, tu seras invité à autoriser.
   - **Jeton Bearer ou clé API :** Si le serveur MCP ne prend pas en charge OAuth, déroule **Advanced settings** et saisis un **bearer token or API key**. Lorsque cette valeur est fournie, Lovable l’utilisera pour authentifier les requêtes au lieu d’OAuth.
4. Clique sur **Add server**.\
   Le serveur MCP personnalisé apparaît maintenant dans ta liste et peut fournir des données contextuelles pour tes builds.

<div id="practical-examples">
  ## Exemples pratiques
</div>

Chaque application connectée devient instantanément disponible comme contexte pour le Lovable Agent. Vous pouvez interagir avec les outils connectés à l’aide d’invites en langage naturel.

| **Scénario**                                          | **Exemple de prompt**                                            | **Exemple de résultat**                                                                     |
| :---------------------------------------------------- | :--------------------------------------------------------------- | :------------------------------------------------------------------------------------------ |
| Un **product manager** connecte **Linear** ou **Jira** | _Génère un prototype à partir des tickets du sprint en cours._   | Lovable lit les descriptions des tickets et les critères d’acceptation pour créer une interface utilisateur alignée et testable. |
| Un **marketer** connecte **Notion** ou **Confluence** | _Transforme notre brief de campagne T4 en landing page._         | Lovable utilise le document comme contexte pour générer une page conforme à la marque et modifiable. |
| Un **ingénieur** connecte **n8n** ou un **MCP interne** | _Crée une interface de workflow qui liste les utilisateurs et permet de les supprimer._ | Lovable crée une application qui utilise des données et des workflows en temps réel.        |

<div id="security-and-permissions">
  ## Sécurité et autorisations
</div>

Les connexions au serveur MCP sont **propres à chaque utilisateur et personnelles**. Vous pouvez consulter ou révoquer les connexions à tout moment dans **Settings → Integrations**.

Pour un accès rapide au niveau du projet, cliquez sur `+` dans la zone de prompt et allez dans **Integrations**.

Avec les abonnements **Business** et **Enterprise**, les **administrateurs et propriétaires** de l’espace de travail peuvent déterminer quels serveurs sont disponibles pour tous les utilisateurs :

- Allez dans **Settings → Privacy & Security** pour activer ou désactiver tout accès aux serveurs MCP pour l’ensemble de l’espace de travail.
- Allez dans **Settings → Integrations → Manage servers** pour activer ou désactiver chaque serveur MCP individuellement.

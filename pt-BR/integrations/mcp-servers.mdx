**MCP (Model Context Protocol)** é um padrão aberto que conecta sistemas de IA, como o Lovable, a ferramentas, serviços e fontes de dados externas.

Ao conectar o Lovable a ferramentas como Notion, Atlassian ou Linear, o Lovable Agent pode usar o contexto real da equipe para gerar código, protótipos e documentação com mais precisão.

Quando você conecta um servidor MCP, o Lovable pode ler dados estruturados (como texto de documentos, issues ou diagramas) e, quando suportado, executar ações limitadas, como criar ou atualizar itens.

O Lovable inclui vários servidores MCP pré‑configurados em todos os planos, com a opção de adicionar servidores personalizados nos planos Business e Enterprise.

<div id="why-connect-mcp-servers">
  ## Por que conectar servidores MCP
</div>

No Lovable, os servidores MCP desbloqueiam duas funcionalidades principais:

- **Traga seu contexto existente para o Lovable.**\
  O Lovable Agent pode ler a documentação da sua equipe, chamados/tickets e arquivos de design para criar protótipos e fluxos alinhados com seus padrões.
- **Conecte seus apps Lovable aos seus fluxos de trabalho.**\
  O Lovable Agent pode executar ações posteriores nas suas ferramentas conectadas — como atualizar o status de chamados/tickets, adicionar links de protótipo como comentários ou criar novos itens quando houver suporte.

O Lovable se torna mais poderoso quando entende o mundo da sua equipe — eliminando suposições, acelerando a iteração e ajudando você a ir de **ideia a app em produção mais rápido do que nunca.**

<div id="how-to-connect-a-prebuilt-mcp-server">
  ### Como conectar um servidor MCP pré‑configurado
</div>

Para usar integrações pré‑configuradas com apps e serviços de terceiros populares:

1. Vá para **Settings → Integrations → MCP servers**.
2. **Selecione um servidor MCP** (por exemplo, Notion, Linear ou Atlassian).
3. Clique em **Connect**. Faça login e conceda à Lovable acesso à sua conta.

Além dessas etapas, você precisa fazer o seguinte no [n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server):

- Vá para **Settings → MCP access** e ative **Enable MCP access** (requer permissões de proprietário da instância ou administrador).
- Para cada workflow que você quiser disponibilizar, vá para **Settings** no editor de workflow e ative **Available in MCP**.

<div id="custom-mcp-servers-business-enterprise">
  ## Servidores MCP personalizados (Business & Enterprise)
</div>

Para ter total flexibilidade, você pode conectar **servidores MCP personalizados** para integrar sistemas internos ou de terceiros ao Lovable — perfeito para APIs privadas, CRMs internos ou fontes de dados personalizadas.

Explore os [servidores MCP remotos](https://mcpservers.org/remote-mcp-servers), uma lista curada de servidores MCP remotos de alta qualidade compatíveis com o Lovable.

<div id="how-to-connect-a-custom-mcp-server">
  ### Como conectar um servidor MCP personalizado
</div>

Para conectar um servidor MCP personalizado ou interno:

1. Acesse **Settings → Integrations → Your MCP servers.**
2. Clique em **Set up** ao lado de **New MCP server**.
3. **Informe os detalhes do servidor:**
   - **Server name**: Um nome claro para o seu servidor (por exemplo, _Internal CRM_ ou _Analytics API_).
   - **Server URL**: O endereço em que o Lovable pode acessar seu servidor MCP.
4. **Authentication:**
   - **OAuth (default):** O Lovable usa OAuth por padrão. Depois de clicar em **Add server**, você será direcionado para autorizar.
   - **Bearer token or API key:** Se o servidor MCP não oferecer suporte a OAuth, expanda **Advanced settings** e insira um **bearer token or API key**. Quando fornecidos, o Lovable usará esses dados para autenticar as solicitações em vez de OAuth.
4. Clique em **Add server**.\
   O servidor MCP personalizado agora aparece na sua lista e pode fornecer dados contextuais para builds.

  ## Exemplos práticos

Cada app conectada fica imediatamente disponível como contexto para o Lovable Agent. Você pode interagir com as ferramentas conectadas usando instruções em linguagem natural.

| **Cenário**                                           | **Exemplo de Prompt de IA**                                      | **Exemplo de resultado**                                                               |

| Um(a) **marketer** conecta **Notion** ou **Confluence**     | _Transforme nosso briefing da campanha de Q4 em uma landing page._ | Lovable usa o documento como contexto para gerar uma página editável e alinhada à marca.      |

<div id="security-and-permissions">
  ## Segurança e permissões
</div>

As conexões de servidor MCP são **vinculadas ao usuário e pessoais**. Você pode revisar ou revogar essas conexões a qualquer momento em **Settings → Integrations**.

Para acesso rápido no nível do projeto, clique em `+` na caixa de Prompt de IA e vá até **Integrations**.

Nos planos **Business** e **Enterprise**, **administradores e proprietários** da área de trabalho podem gerenciar quais servidores ficam disponíveis para todos os usuários:

- Vá até **Settings → Privacy & Security** para ativar ou desativar todo o acesso a servidores MCP para toda a área de trabalho.
- Vá até **Settings → Integrations → Manage servers** para ativar ou desativar servidores MCP individuais.

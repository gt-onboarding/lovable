Lovable tem integrações nativas que permitem exportar código e adicionar um backend. Também é possível integrar uma API externa se você quiser adicionar uma funcionalidade específica ao seu site.

<div id="what-is-an-api">
  ## O que é uma API?
</div>

Uma API é uma forma de aplicativos de software se comunicarem e compartilharem dados entre si.
APIs diferentes oferecem diferentes funcionalidades, como enviar e-mails ou processar pagamentos.
Você pode ler mais sobre o poder das APIs [neste artigo em nosso blog](https://lovable.dev/blog/2025-01-28-the-power-of-apis).

<div id="using-any-api-with-lovable">
  ## Usando qualquer API com o Lovable
</div>

Todas as outras APIs que podem ser chamadas ainda podem ser integradas, mas muitas vezes exigem contexto e instruções adicionais.

Você pode ter:

* **Sistemas internos com APIs personalizadas**\
  Esses podem ser serviços que sua equipe criou (por exemplo, sistemas de inventário, pipelines de analytics, ferramentas internas) que expõem endpoints REST ou GraphQL, mas que não são documentados publicamente ou amplamente conhecidos.
  Você quer que o Lovable consuma ou envie dados para esses sistemas para automatizar fluxos de trabalho ou alimentar interfaces de usuário.
* **APIs de terceiros que o Lovable ainda não suporta**\
  Talvez você use um produto SaaS de nicho, um provedor de dados especializado ou uma integração com um parceiro que o Lovable ainda não conhece nativamente.
  Mesmo que não seja oficialmente compatível, você ainda quer que o Lovable se conecte, faça chamadas e trabalhe com esses dados.

Nos dois casos, o desafio é o mesmo: o Lovable não “conhece” a API por padrão. Mas, com boa documentação e uma configuração de autenticação adequada, ele ainda pode usá-la de forma eficaz.
De qualquer forma, a abordagem é a mesma: **o Lovable ainda pode se conectar a ela**. Veja como.

<Steps>
  <Step title="Dê ao Lovable o contexto de que ele precisa">
    O Lovable funciona melhor quando entende a API. Você pode fornecer:

    * **Especificação OpenAPI** (se disponível)
    * **Documentação ou exemplos copiados e colados** (especialmente endpoints e exemplos de requisições/respostas)

    Quanto mais clara a documentação, melhor o Lovable consegue raciocinar sobre como chamar a API.
  </Step>

  <Step title="Trate a autenticação com segurança">
    Para APIs que exigem autenticação:

    **Opção 1: Use Supabase Edge Functions como proxy**

    * Armazene segredos com segurança no Supabase
    * Deixe o Lovable chamar o endpoint da API por meio da Função de borda (e não a API diretamente)

    **Opção 2: Implemente a integração com a API diretamente no seu Projeto Lovable**

    * Exponha um endpoint de autenticação (por exemplo, para trocar credenciais por um token ou sessão)
    * Deixe o Lovable chamar esse endpoint de autenticação como parte do fluxo de integração
    * Inclua tokens, cabeçalhos ou informações de sessão nas chamadas de API subsequentes
  </Step>
</Steps>

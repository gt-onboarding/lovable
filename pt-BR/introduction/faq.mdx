<div id="getting-started">
  ## Primeiros passos
</div>

<AccordionGroup>
  <Accordion title="O que é a Lovable?">
    Lovable é uma plataforma baseada em IA que permite que usuários de qualquer nível de experiência criem sites full‑stack usando linguagem natural. Basta descrever o que você quer e a Lovable constrói para você.
  </Accordion>

  <Accordion title="O que posso construir com a Lovable?">
    Se você consegue imaginar, você consegue construir com a Lovable. De landing pages simples a ferramentas poderosas voltadas para casos de uso específicos, a Lovable dá a você a flexibilidade para criar o que precisar, sem limites.
  </Accordion>

  <Accordion title="Preciso ter experiência em programação para construir com a Lovable?">
    Não. A Lovable foi criada para todos os níveis de experiência. Seja você uma pessoa desenvolvedora ou alguém não técnico, você pode criar sites usando a Lovable.
  </Accordion>

  <Accordion title="Como crio um projeto?">
    Para criar um projeto, basta digitar uma mensagem na caixa de mensagem do dashboard. Descreva o que você quer construir e a Lovable assume a partir daí, gerando um ponto de partida para o seu projeto. Lembre-se de começar pequeno e construir em etapas!
  </Accordion>

  <Accordion title="Posso reutilizar um projeto existente?">
    Sim, você pode remixar um projeto público existente ou um dos seus. Remixar um projeto permite que você reutilize o estado atual dele como ponto de partida e construa em cima disso. É uma ótima maneira de explorar novas ideias, fazer ajustes ou iterar com mudanças diferentes enquanto preserva a versão original.
  </Accordion>
</AccordionGroup>

<div id="building-with-lovable">
  ## Construindo com a Lovable
</div>

<AccordionGroup>
  <Accordion title="Como posso construir de forma eficiente com a Lovable?">
    Construir de forma eficiente com a Lovable é uma questão de usar as técnicas certas. Qualquer pessoa consegue com um pouco de prática. Para ajudar você a começar, confira nosso [guia](https://mastering-lovable.lovable.app/).
  </Accordion>

  <Accordion title="Como devo estruturar meus prompts para ter os melhores resultados?">
    * Seja claro e específico.
    * Foque em um recurso por vez.
    * Construa de forma incremental.
    * Use palavras de efeito para acertar no design.
  </Accordion>

  <Accordion title="O que devo fazer quando encontrar um erro?">
    <Steps>
      <Step title="Botão Try-to-fix">
        Use a opção **Try-to-fix**. A Lovable tentará resolver o problema automaticamente. Usar o botão não desconta nenhum crédito.
      </Step>

      <Step title="Investigar">
        Se **Try-to-fix** não resolver o problema, peça para o Lovable Agent investigar a questão. Explique o que você está vendo e quais partes específicas não estão funcionando.
      </Step>

      <Step title="Reverter para uma versão anterior">
        Se isso não funcionar, você pode visualizar uma versão anterior que esteja funcionando e depois reverter para essa versão, se necessário, ou reverter e editar uma mensagem passada para explorar uma abordagem diferente. Nada é perdido — todas as alterações feitas depois daquele ponto permanecem no chat e podem ser reaplicadas a qualquer momento.
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="O que é refatoração e por que ela é importante?">
    Refatoração melhora a estrutura e a eficiência do código sem alterar a funcionalidade. Ela ajuda a:

    * Otimizar o desempenho removendo redundâncias.
    * Melhorar a capacidade de manutenção para futuras atualizações.
    * Evitar conflitos estruturando a lógica de forma mais clara.
  </Accordion>

  <Accordion title="Posso conectar meu projeto a um backend para armazenar dados?">
    Se você precisa adicionar recursos de backend à sua aplicação, você pode:

    * Conectar ao backend nativo da Lovable – [Lovable Cloud](https://docs.lovable.dev/features/cloud)
    * Usar a [integração nativa com Supabase](https://docs.lovable.dev/integrations/supabase) da Lovable
  </Accordion>

  <Accordion title="Posso adicionar login ao meu site?">
    Sim, você pode adicionar autenticação usando o [Lovable Cloud](https://docs.lovable.dev/features/cloud) ou integrando com o [Supabase](https://docs.lovable.dev/integrations/supabase). Basta pedir em um prompt de IA para a IA adicionar isso e seguir as instruções.
  </Accordion>

  <Accordion title="Posso adicionar APIs de terceiros ao meu projeto?">
    Sim, você pode integrar APIs como OpenAI, Stripe e outras, especificando-as no seu prompt de IA. Leia mais [aqui](/pt-BR/integrations/introduction).
  </Accordion>

  <Accordion title="Posso armazenar chaves de API confidenciais na Lovable?">
    Não! Nunca insira suas chaves de API diretamente na Lovable. Se você estiver integrando com uma API, recomendamos usar segredos no [Lovable Cloud](https://docs.lovable.dev/features/cloud) ou no [Supabase](https://docs.lovable.dev/integrations/supabase) para armazenar suas chaves de API, em combinação com as Edge Functions deles.
  </Accordion>

  <Accordion title="Como adiciono pagamentos ao meu site?">
    Você pode adicionar pagamentos adicionando o Stripe ao seu projeto. Veja [integração com Stripe](/pt-BR/integrations/stripe) para configurá-la.
  </Accordion>

  <Accordion title="Como posso melhorar o SEO do meu projeto?">
    Garanta os melhores resultados seguindo nosso guia [Implementar boas práticas de SEO e GEO](https://docs.lovable.dev/tips-tricks/seo-geo).
  </Accordion>

  <Accordion title="Quanto tempo leva para concluir um projeto?">
    Depende da complexidade. Ao dividir as tarefas em etapas menores, planejar com antecedência e refinar com base no feedback, você consegue avançar rapidamente garantindo a qualidade.
  </Accordion>
</AccordionGroup>

<div id="features">
  ## Funcionalidades
</div>

<AccordionGroup>
  <Accordion title="Como faço para acessar as configurações do Projeto?">
    Você pode encontrar as **Configurações do Projeto** clicando no nome do seu projeto no canto superior esquerdo do editor e, em seguida, selecionando **Configurações**.
  </Accordion>

  <Accordion title="Posso alterar a visibilidade do meu Projeto?">
    Sim, vá até as **Configurações do Projeto**, onde você pode alterar a visibilidade. Consulte [Controlar a visibilidade do Projeto](/pt-BR/features/private-projects) para mais informações.
  </Accordion>

  <Accordion title="Posso solicitar acesso a projetos privados?">
    Sim. Ao acessar um projeto privado, você pode solicitar acesso como visualizador ou editor ao proprietário do projeto. Isso mantém o trabalho privado seguro e, ao mesmo tempo, facilita a colaboração.
  </Accordion>

  <Accordion title="Posso ocultar o selo da Lovable?">
    Sim, vá até as **configurações do Projeto**, onde você encontrará a opção para ocultar o selo. Somente usuários pagantes podem desativar o selo.
  </Accordion>

  <Accordion title="Posso renomear um projeto?">
    Sim. Acesse as **Configurações do projeto**, onde você encontrará a opção para renomear o projeto.
  </Accordion>

  <Accordion title="Posso duplicar um projeto?">
    Para fazer uma cópia de um projeto, você precisa remixar o projeto. Acesse **Project Settings** e selecione a opção **Remix**. Isso vai criar uma cópia exata do projeto na área de trabalho atualmente selecionada.

    <Note>
      Você só pode remixar projetos públicos de outros usuários se eles não estiverem conectados ao Supabase.
    </Note>
  </Accordion>

  <Accordion title="Posso excluir um projeto?">
    Sim, acesse as **Configurações do Projeto**, e lá você encontrará a opção **Excluir** o projeto.

    <Note>
      Esta ação é definitiva e não há como restaurar projetos excluídos.
    </Note>
  </Accordion>

  <Accordion title="O que é um Remix?">
    Remixar um projeto cria uma cópia idêntica que você pode editar separadamente. Saiba mais [aqui](/pt-BR/introduction/faq#can-i-make-a-copy-of-a-project).
  </Accordion>

  <Accordion title="O que é o Chat mode?">
    Você pode ler mais sobre os diferentes modos do Lovable e o Chat mode [aqui](/pt-BR/features/modes).
  </Accordion>

  <Accordion title="Posso ver o código que o Lovable gera e editá‑lo manualmente?">
    Sim. Você pode visualizar e editar manualmente o código usando o [Code mode](/pt-BR/features/code-mode) da Lovable.
  </Accordion>

  <Accordion title="Posso editar manualmente o texto ou cores?">
    Sim, você pode editar textos ou cores usando o recurso [Visual Edits](/pt-BR/features/visual-edit).
  </Accordion>

  <Accordion title="Posso colaborar nos meus projetos?">
    Sim, você pode colaborar em projetos no Lovable. Consulte [Colaboração](/pt-BR/features/collaboration) para mais informações.
  </Accordion>

  <Accordion title="Como publico meu projeto?">
    Consulte [Publicar](/pt-BR/features/publish) e siga as etapas indicadas ali para publicar o seu Projeto.
  </Accordion>

  <Accordion title="Como faço para adicionar um domínio personalizado ao meu projeto?">
    Consulte [Domínio personalizado](/pt-BR/features/custom-domain) para adicionar um domínio personalizado ao seu projeto.
  </Accordion>

  <Accordion title="Posso ver métricas dos meus projetos já publicados?">
    Sim. Consulte [análises do Projeto](/pt-BR/features/project-analytics) para mais informações.
  </Accordion>

  <Accordion title="Posso adicionar imagens a um Prompt de IA no Lovable?">
    Sim. Imagens usadas como inspiração de design, esboços do layout desejado ou capturas de tela de problemas na interface são muito úteis durante o processo de criação do seu site.
  </Accordion>

  <Accordion title="Posso adicionar vídeos a um Prompt de IA no Lovable?">
    Infelizmente, vídeos ainda não são suportados, mas capturas de tela geralmente resolvem.
  </Accordion>

  <Accordion title="Posso exportar todo o código do meu projeto?">
    Sim. Ao transferir seu código para o GitHub, você pode exportá-lo e fazer o que quiser com ele. Veja [integração com GitHub](/pt-BR/integrations/github) para saber mais.
  </Accordion>

  <Accordion title="Posso começar um projeto importando código de uma fonte externa, como o GitHub?">
    Não, no momento não é possível iniciar um Projeto no Lovable a partir de código já existente, por exemplo, no GitHub.
  </Accordion>

  <Accordion title="Como posso ver o histórico do meu projeto?">
    Você pode ver o histórico do seu projeto simplesmente rolando a conversa para cima ou clicando no ícone **View history** na parte superior do chat. Você também pode selecionar versões mais antigas para pré-visualizá-las ou restaurar seu projeto para essa versão.
  </Accordion>

  <Accordion title="Posso voltar para uma versão anterior do meu projeto?">
    Sim. Você pode visualizar uma versão anterior que estava funcionando e, se necessário, reverter para essa versão, ou editar uma mensagem passada e reverter para explorar uma abordagem diferente. Nada é perdido — todas as alterações feitas depois daquele ponto permanecem no chat e podem ser reaplicadas a qualquer momento.

    * Você encontra a opção de reverter em cada resposta no chat ou na aba de histórico, ao selecionar a versão para a qual quer reverter.
    * Você encontra a opção de editar uma mensagem passada e reverter em cada mensagem antiga que enviou no chat.
  </Accordion>
</AccordionGroup>

Para dúvidas específicas sobre recursos e para ver dicas de solução de problemas, confira as [páginas](/pt-BR/features) de cada recurso.

<div id="lovable-api">
  ## API do Lovable
</div>

<AccordionGroup>
  <Accordion title="O que posso fazer com a API do Lovable?">
    A **API do Lovable** permite criar e compartilhar aplicativos Lovable de forma programática — transformando ideias, instruções ou até mesmo imagens em aplicações completas geradas a partir de um único link. 

    A primeira versão, **Build with URL**, permite gerar aplicativos diretamente a partir de uma URL compartilhável. Com ela, você pode:

    * Incorporar um botão **Build with Lovable** no seu site ou aplicativo.
    * Compartilhar instruções predefinidas via URLs com amigos, colegas ou nas redes sociais.
    * Automatizar a geração de aplicativos como parte de fluxos de trabalho internos.
    * Criar protótipos de aplicativos rapidamente usando URLs com instruções e imagens de referência.

    Saiba mais sobre o recurso e casos de uso reais no nosso post de lançamento: [Build apps from a link with Lovable Build with URL](https://lovable.dev/blog/introducing-lovable-api-build-with-url).

    Leia o guia técnico completo aqui: [Lovable API: Build with URL documentation](https://docs.lovable.dev/integrations/build-with-url).
  </Accordion>
</AccordionGroup>

<div id="managing-your-account">
  ## Gerenciando sua conta
</div>

<AccordionGroup>
  <Accordion title="Posso alterar o e-mail que uso para fazer login?">
    Infelizmente, alterar o endereço de e-mail diretamente não é possível no momento. Como alternativa, você pode criar uma nova conta com o e-mail desejado e transferir seus projetos seguindo estas etapas:

    1. Torne seus projetos existentes públicos temporariamente e copie a URL.
    2. Acesse a URL enquanto estiver logado na sua nova conta.
    3. Remixe os projetos na sua nova conta. Observe que projetos conectados ao Supabase precisam ser desconectados do Supabase antes que você possa remixá-los.
    4. Depois que os projetos forem transferidos, você poderá torná-los privados novamente, se necessário.

    Observação: Se a sua área de trabalho de destino estiver em um plano pago, pode ser necessário remixar projetos públicos para conseguir acessá-los. Projetos remixados não levarão consigo recursos do nível de assinatura, a menos que a nova área de trabalho também os tenha.
  </Accordion>

  <Accordion title="Posso excluir minha conta?">
    Para excluir sua conta, vá em **Settings → Your Account**, role até o final e clique em **Delete account**. Isso removerá permanentemente sua conta e todos os dados associados.
  </Accordion>

  <Accordion title="Posso reativar uma conta excluída?">
    Não, você não pode reativar uma conta excluída. No entanto, após clicar no botão para excluir a conta, essa ação pode ser cancelada em até 48 horas para interromper a exclusão.
  </Accordion>

  <Accordion title="Posso alterar o proprietário da área de trabalho?">
    Sim, um proprietário pode adicionar vários proprietários da área de trabalho.
  </Accordion>

  <Accordion title="Como posso transferir a propriedade de um projeto para outra pessoa?">
    Uma forma de fazer isso é remixar o projeto; a pessoa que remixar o projeto passará a ser a proprietária daquela versão.
  </Accordion>
</AccordionGroup>

Para outras dúvidas sobre créditos e problemas de assinatura, consulte [Planos e créditos](/pt-BR/introduction/plans-and-credits).

<div id="how-lovable-works">
  ## Como o Lovable funciona
</div>

<AccordionGroup>
  <Accordion title="Quais stacks de tecnologia o Lovable usa?">
    O Lovable cria o front-end usando React, Tailwind e Vite, e pode se conectar a backends compatíveis com OpenAPI. Há suporte para persistência de dados e autenticação usando Lovable Cloud ou Supabase.
  </Accordion>

  <Accordion title="O Lovable oferece suporte ao desenvolvimento de aplicativos móveis?">
    Não, o Lovable é focado em **aplicações web**, mas você pode criar aplicativos web adaptados para dispositivos móveis.
  </Accordion>

  <Accordion title="Como o Lovable lembra do contexto?">
    O Lovable processa as mensagens recentes para manter o contexto. Para obter os melhores resultados, inclua detalhes relevantes em cada prompt de IA para garantir clareza e precisão.
  </Accordion>

  <Accordion title="Quem é o dono dos projetos e do código que o Lovable cria?">
    Você, como criador, é o dono!
  </Accordion>
</AccordionGroup>

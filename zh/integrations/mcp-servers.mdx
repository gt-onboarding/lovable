**MCP（Model Context Protocol）** 是一种开放标准，用于将 Lovable 等 AI 系统连接到外部工具、服务和数据源。

通过将 Lovable 连接到 Notion、Atlassian 或 Linear 等工具，Lovable Agent 可以利用你团队的真实上下文来生成更准确的代码、原型和文档。

当你连接 MCP 服务器后，Lovable 可以读取结构化数据（如文档、Issue 或图表中的文本），并在受支持的情况下执行有限的操作，例如创建或更新项目。

所有套餐中都包含若干预置的 MCP 服务器，Business 和 Enterprise 套餐还支持添加自定义 MCP 服务器。

<div id="why-connect-mcp-servers">
  ## 为什么要连接 MCP 服务器
</div>

在 Lovable 中，MCP 服务器带来两个关键能力：

- **把你现有的上下文带入 Lovable。**\
  Lovable Agent 可以读取你团队的文档、工单和设计文件，以符合你标准的方式构建原型和流程。
- **让你的 Lovable 应用连接到你的工作流程。**\
  Lovable Agent 可以在你已连接的工具中执行后续操作——例如更新工单状态、将原型链接添加为评论，或在支持的工具中创建新条目。

当 Lovable 理解你团队的世界时，它会变得更强大——减少盲猜、加快迭代，帮助你以前所未有的速度从**想法走向上线应用。**

<div id="how-to-connect-a-prebuilt-mcp-server">
  ### 如何连接预构建的 MCP 服务器
</div>

要使用针对常见第三方应用和服务的预构建集成：

1. 前往 **Settings → Integrations → MCP servers**。
2. **选择一个 MCP 服务器**（例如 Notion、Linear 或 Atlassian）。
3. 点击 **Connect**，登录并授予 Lovable 访问你账号的权限。

除了上述步骤，你还需要在 [n8n](https://docs.n8n.io/advanced-ai/accessing-n8n-mcp-server) 中进行以下操作：

- 前往 **Settings → MCP access** 并启用 **Enable MCP access**（需要实例所有者或管理员权限）
- 对于每个你想公开的工作流，在工作流编辑器中打开 **Settings** 并启用 **Available in MCP.**

<div id="custom-mcp-servers-business-enterprise">
  ## 自定义 MCP 服务器（Business 与 Enterprise）
</div>

为了获得充分的灵活性，你可以连接**自定义 MCP 服务器**，将内部或第三方系统接入 Lovable——非常适合私有 API、内部 CRM 或自定义数据源。

查看[远程 MCP 服务器](https://mcpservers.org/remote-mcp-servers)，这是一个与 Lovable 兼容的高质量远程 MCP 服务器精选列表。

<div id="how-to-connect-a-custom-mcp-server">
  ### 如何连接自定义 MCP 服务器
</div>

要连接自定义或内部 MCP 服务器：

1. 前往 **Settings → Integrations → Your MCP servers.**
2. 点击 **Set up**（位于 **New MCP server** 旁边）。
3. **输入服务器详细信息：**
   - **Server name**：为你的服务器设置一个清晰的名称（例如 _Internal CRM_ 或 _Analytics API_）。
   - **Server URL**：Lovable 访问你的 MCP 服务器所使用的地址。
4. **Authentication：**
   - **OAuth (default)：** Lovable 默认使用 OAuth。点击 **Add server** 后，系统会提示你完成授权。
   - **Bearer token or API key：** 如果 MCP 服务器不支持 OAuth，展开 **Advanced settings** 并输入 **bearer token 或 API 密钥**。提供后，Lovable 将使用此凭证对请求进行身份验证，而不再使用 OAuth。
4. 点击 **Add server**。\
   自定义 MCP 服务器现在会出现在你的列表中，并可为构建过程提供上下文数据。

<div id="security-and-permissions">
  ## 安全性与权限
</div>

MCP 服务器连接是**基于用户、仅供个人使用**的。你可以随时在 **Settings → Integrations** 中查看或撤销连接。

若要在项目层面快速访问，请点击提示框中的 `+`，然后前往 **Integrations**。

在 **Business** 和 **Enterprise** 方案中，工作区的 **管理员和所有者** 可以管理哪些服务器对所有用户可用：

- 前往 **Settings → Privacy & Security**，为整个工作区启用或禁用所有 MCP 服务器访问。
- 前往 **Settings → Integrations → Manage servers**，启用或禁用单个 MCP 服务器。

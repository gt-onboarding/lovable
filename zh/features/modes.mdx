在 Lovable 中发送消息时，你可以根据不同的用途选择不同的模式。

<div id="agent-mode">
  ## Agent 模式
</div>

Agent 模式让 Lovable 能够自主思考、规划和执行操作。它旨在让 Lovable 在处理复杂开发任务时更加强大、可靠并且独立。
Agent 模式可以：

* 理解你的请求
* 在你的代码库中探索并获取所需上下文
* 按需读取文件
* 根据需要修复和重构代码
* 主动调试问题
* 检查日志和网络活动，以自动发现并修复错误
* 实时搜索网络以获取文档、素材或截图
* 为你的应用生成和编辑图片

以上全部都无需人工干预。通过对代码库的更深入理解和实时调试，它能够在减少错误和不必要改动的同时，产出更好的结果。
这些工具让 Lovable 能够更自主、更精确地行动，从而减少开发过程中的摩擦。

**计费**

Agent 模式采用按量计费，你可以在[这里](/zh/introduction/plans-and-credits#credit-usage)了解更多。每条消息的费用取决于其复杂度，确保你只为真正使用到的部分付费。
许多消息的费用低于 1 个积分，更复杂的消息可能会花费更多积分。

[点击这里阅读公告\！](https://lovable.dev/blog/agent)

<div id="chat-mode">
  ## Chat mode
</div>

Chat mode 是你的开发和调试伙伴。它能帮助你思考问题、调试故障，并直接在 Lovable 中规划你的产品。
在 Chat mode 中发送的每条消息都会消耗 1 个额度。

<Card title="启用 Chat mode" img="/assets/features/images/chat-mode-edit.png">
  要启用 **Chat mode**，只需点击你在 Lovable 中输入消息的位置旁边的 “Chat” 按钮。
</Card>

与 Agent mode 不同，Chat mode 是对话式的，不能直接修改你的代码。它非常适合用于提问、规划你的项目和调试。
它具备智能代理能力，也就是说，它可以进行多步推理，并自行决定何时搜索文件、检查日志、查询数据库等。

如果你请它帮助实现某个功能或进行故障排查，它会提出一个循序渐进的计划，帮助你达成期望的结果。
你会看到一个 “Implement the plan” 按钮提示。只需点击它，AI 就会进入 Agent mode 并实现所需的代码更改。

**Chat mode 的最佳使用场景**

* 在请求修复之前，先让 AI 深入排查问题，从而实现高效调试。
* 规划数据库结构和关系，以支持高效的系统架构。
* 通过识别实现新功能所需的最小变更，以最优方式实现新功能。
* 请求改进建议，以提升你的应用。
* 理解并评估潜在新功能将如何影响应用的结构和功能。

Lovable Cloud 是一个全栈云平台，覆盖你应用的方方面面——从用户看到的前端界面，到在幕后提供支持的后端服务。你只需在 Lovable 中直接描述你想要的效果，Lovable 就会自动生成前端和后端，并准备好在云端运行。你无需再管理硬件或复杂的基础设施，而是可以按需使用托管、计算和存储资源，并在应用增长时自动扩展。

Lovable Cloud 构建在 Supabase 的开源基础之上，这意味着从第一天起，你就拥有一个可直接用于生产环境的运行环境，而无需单独配置 Supabase。你将拥有生产级数据库、实时更新、用户认证和存储能力——且完全不需要任何手动配置步骤。

<div id="why-choose-lovable-cloud">
  ## 为什么选择 Lovable Cloud
</div>

- **随业务增长而扩展**：从第一个原型到数百万用户，Lovable 都能根据你的需求顺畅扩展。
- **开箱即安全**：企业级身份验证、细粒度访问控制以及合规最佳实践，全方位保护你的应用和数据。
- **高性价比**：在一个工作区中管理多个项目，并按用量计费。每个工作区都从每月 **\$25 的免费用量** 起步，足以启动并运行多个项目（临时优惠截至 2025 年底，具体内容可能调整）。在 [Cloud 和 AI 基于用量的定价](https://docs.lovable.dev/features/cloud#usage-based-cloud-and-ai-pricing) 中了解更多。

<div id="enabling-lovable-cloud">
  ## 启用 Lovable Cloud
</div>

默认情况下，你的工作区会启用 Lovable Cloud。你可以在用户偏好设置中管理各项目的 Lovable Cloud 行为：**Settings → Integrations → Lovable Cloud → User preferences.**

<div id="user-preferences">
  ### 用户偏好设置
</div>

Cloud 集成的默认设置为 **每次询问**，这意味着你可以为每个项目决定是否启用或禁用 Cloud。你可以随时在 **Settings → Integrations → Lovable Cloud → User preferences** 中更改你的首选项。

当某个功能需要后端功能时（例如身份验证或文件存储），Lovable 会向你发出提示。你有三个选项：

- **Allow**：为当前项目启用该集成。
- **Deny**：拒绝此次请求的集成（之后可能会再次询问你）。
- **Adjust preferences**：更改未来项目的默认行为（不会影响当前项目）。

<div id="configuring-lovable-cloud-tools">
  ### 配置 Lovable Cloud 工具
</div>

在你的用户偏好设置中，每个 Cloud 工具都有三个选项：

- **Always allow**：Lovable 会自动执行该操作，无需你确认或批准。
- **Ask each time**：每当需要执行该操作时，Lovable 都会请求你的批准。
- **Never allow**：Lovable 会阻止该操作，告知你需要 Cloud 访问权限，并提示你启用 Cloud。

你可以随时在 **Settings → Integrations → Lovable Cloud → User preferences** 中更改 Cloud 工具的偏好设置。

在 Cloud 选项卡中，你可以使用以下功能和设置：

<div id="database">
  ### 数据库
</div>

- **自动生成 schema**：用自然语言描述你需要的数据，Lovable 会为你创建相应的表和 schema。
- **无需 SQL 即可管理**：直接在 UI 中查看、编辑和组织记录。更新字段值、检查表结构，并在不编写查询语句的情况下保持应用的数据同步。

<div id="users-auth">
  ### 用户与身份验证
</div>

- **用户管理**：查看用户账户、跟踪用户活动并管理访问控制。
- **内置身份验证**：Lovable 可以为你的应用生成注册/登录页面，支持邮箱、手机号和 Google 登录。
- **默认安全**：身份验证由 Lovable Cloud 提供支持，确保用户凭证安全。

<div id="storage">
  ### 存储
</div>

- **无需写代码即可处理文件**：上传、管理和访问头像、文档、视频或静态资源等文件。
- **存储桶**：文件安全地存放在项目的存储桶中，你可以在应用中使用对应的 URL 或引用。
- **限制**：文件大小上限为 2 GB。

<div id="edge-functions">
  ### Edge Functions
</div>

- **无服务器逻辑**：为 API、工作流或高级功能运行自定义代码，而无需配置和管理服务器。函数会随流量自动伸缩。

**常见用例：**

- AI 驱动功能，例如 AI 摘要或 AI 聊天机器人。使用内置的 [Lovable AI](https://docs.lovable.dev/features/ai)。
- 发送电子邮件或推送通知
- 支付处理，例如使用 [Stripe](https://docs.lovable.dev/integrations/stripe#stripe-integration)
- 定时任务
- 密集计算或外部 API 调用

**其他亮点：**

- **易于创建**：只需在 Lovable 聊天中描述你需要的函数。
- **内置调试**：每个函数都包含执行、输出和错误的日志。如果出现故障，Lovable 会将错误展示出来，方便你快速排查问题。

<div id="ai">
  ### AI
</div>

- **内置 AI 功能**：Lovable Cloud 内置 AI，你可以快速为应用创建功能，让应用更加智能、更具吸引力。更多信息参见 [Lovable AI](https://docs.lovable.dev/features/ai)。
- 一些由 AI 驱动的功能示例包括：
  - **AI 总结** – 自动将长文本精炼为清晰的要点
  - **AI 聊天机器人或 Agent** – 在应用中构建对话式助手
  - **情感分析** – 大规模理解用户反馈
- 你可以在 AI 概览中监控 Lovable AI 在项目中的使用情况和性能。

<div id="secrets">
  ### 机密信息
</div>

- **安全的环境变量**：用于存储 API 密钥、令牌或凭证等敏感数据，而无需在代码中明文暴露。
- **自动检测**：当某个功能需要机密信息时，Lovable 会通过安全界面提示你输入。
- **加密且安全**：机密信息会被安全存储，并自动注入到 Edge Functions 或其他集成中使用。

**示例：** 只需保存一次你的 Stripe Secret Key，Lovable 就会确保在支付处理函数中自动使用它，而无需你在代码中进行硬编码。

<Note>
  某些机密信息（例如 Supabase 密钥和 Lovable API 密钥）会自动添加，并且是 Lovable 平台正常运行所必需的。这些密钥无法删除。
</Note>

<div id="logs">
  ### 日志
</div>

- **实时监控**：实时查看应用的活动。
- **调试和追踪问题**：识别错误、跟踪事件，并确保你的后端运行健康。

<div id="advanced-settings-upgrade-instance">
  ### 高级设置 - 升级实例
</div>

使用高级设置来查看和管理你当前的实例规格。更大的实例可以让你的应用处理更多用户和流量。请记住，随着应用使用量的增长，你在 Cloud 上的使用成本也会随之增加。

- **免费套餐用户**：可随时通过升级套餐，将实例从 **Tiny** 升级到更大的规格
- **付费套餐用户**：可从下面的选项中选择新的实例规格：
  - **Tiny**：非常适合尝试和实验
  - **Mini**：适合早期项目的可靠选择
  - **Small**：为你的应用预留成长空间
  - **Medium**：适合日常使用的稳定选择
  - **Large**：适合高需求场景的放心选择

<div id="example-flow-prompting-lovable-to-build-a-crm-tool">
  ## 示例流程 - 使用提示让 Lovable 构建一个 CRM 工具
</div>

_帮我构建一个 CRM 工具，用来管理客户、跟进交易、处理发票，并有条理地保存笔记。界面设计保持简洁、现代、商务风格，并包含一个用于导航的侧边栏。_

<div id="step-1-organize-the-data-database">
  ### 步骤 1：整理数据（数据库）
</div>

告诉 Lovable 这个工具需要帮你记录哪些信息。Lovable 会自动为你创建数据表，所以你不需要自己去操作数据库。

_我想要有个地方来记录我的客户信息，包括姓名、电子邮件、电话号码和备注。我还需要一种方式来跟踪交易，包括对应哪位客户、交易金额、所处阶段，以及我预计的成交时间。还需要一个发票部分，包括对应的客户、金额、状态和到期日。_

<div id="step-2-make-it-secure-users-auth">
  ### 步骤 2：保障安全（用户与认证）
</div>

添加登录功能，让数据保持私密，并且用户可以轻松登录。

_为应用添加登录功能。在访问 CRM 仪表盘之前要求用户先登录。同时，添加一个「使用 Google 登录」按钮，让用户可以用自己的 Google 账号登录。_

<div id="step-3-handle-files-storage">
  ### 步骤 3：处理文件（存储）
</div>

你可以让用户在应用内直接上传和查看文件。Lovable 会负责存储和关联，你只需说明哪些文件应该放在哪里。

_为每个商机添加一个合同上传字段。让客户设置个人头像，并在客户详情页中展示。上传发票 PDF，并在发票列表中显示下载链接。_

<div id="step-4-automate-workflows-edge-functions">
  ### 步骤 4：自动化工作流 (Edge Functions)
</div>

设置自动化操作，为你节省时间。

_当 Clients 表中新增潜在客户时，向我发送邮件通知。使用 Lovable AI 分析客户会议记录，并将情感评分保存到 Clients 表中。自动同步新联系人。安排一个每天早上 7 点运行的任务，向我发送当天跟进事项摘要邮件。当一笔付款在 Stripe 中被标记为“paid”时，自动将相关发票的状态更新为“Paid”。_

<div id="step-5-connect-to-other-tools-secrets">
  ### 步骤 5：连接其他工具（机密信息）
</div>

当你使用 Stripe 或 Google Calendar 之类的服务时，它们会给你专属的“密钥”。\
Lovable 会在需要时提示你以安全的方式添加这些信息，并且只有你的应用可以使用它们。

<div id="step-6-add-ai-superpowers-ai">
  ### 第 6 步：添加 AI 超能力（AI）
</div>

让这个工具像你的助理一样工作。

_总结每位客户的会议记录并保存到他们的个人资料中。为每个进行中的交易提出下一步建议。为跟进潜在客户生成个性化邮件草稿。将客户消息归类为：销售、客户支持或账单。_

<div id="usage-based-cloud-and-ai-pricing">
  ## 基于用量的 Cloud 和 AI 定价
</div>

Lovable 在所有定价方案中都包含 Cloud 托管和内置 AI，甚至包括免费方案。这些服务采用**基于用量的定价模型**。也就是说，你的费用会随使用量增长，不包含在订阅费用中。

在你的工作区或项目中，前往 **Settings → Cloud & AI balance**。在这里，你可以：

- 查看工作区余额状态
- 为工作区余额充值并添加资金（仅适用于付费方案）
- 查看发票历史并管理支付偏好设置
- 跟踪整个工作区的 Cloud 和 AI 总使用成本
- 查看每个项目的详细 Cloud 和 AI 使用成本

  <Note>
    目前，该列表会包含所有项目，即使这些项目未使用 Lovable Cloud 或 AI。没有使用记录的项目始终显示 \$0.00。
  </Note>

Cloud & AI balance 页面会按以下方式更新：

- Cloud 使用情况每天在中欧时间上午 10 点更新
- AI 使用情况持续更新

<Note>
  - 你的 **Cloud balance** 用于支付已部署应用的托管费用。
  - 你的 **AI balance** 用于支付已部署应用中使用的 AI 功能费用。
  - **Workspace Credits** 是分开的，仅用于在构建应用时发送消息，并可在 **Settings → Plans & credits** 中查看和跟踪。
</Note>

<div id="included-free-usage-all-plans">
  ### 包含的免费用量（适用于所有套餐）
</div>

无论你选择哪种套餐，也无论你在当月的何时开始订阅，每个工作区每月都会获得免费用量：

- **\$25 Cloud 余额**（用于应用托管）
- **\$1 AI 余额**（用于 AI 功能）

这些免费额度会在每个月的第一天 00:00 UTC（会根据你的时区显示）自动重置，并且与订阅套餐的计费周期无关。未使用的免费额度不会结转到下个月。

<Note>
  **限时优惠，可能调整：** 截至 2025 年底，每个工作区每月都会获得 \$25 Cloud 和 \$1 AI，即使是 Free 套餐用户也同样适用。
</Note>

<div id="adding-funds-paid-plans-only">
  ### 添加资金（仅适用于付费方案）
</div>

只有在付费方案中，你才能为 Lovable Cloud 和 AI 使用添加工作区余额。免费方案用户需要先升级才能使用此功能。

任何超出包含额度的用量都会单独计费，不包含在你的订阅内。

- **Cloud 使用：** 费用取决于活动量（用户数量和交互次数）以及数据传输量（上传/下载）。
- **AI 使用：** 费用取决于所用模型以及任务复杂度（模型越高级、任务越复杂，费用越高）。要核实费用，请参考我们在 [Lovable AI](https://docs.lovable.dev/features/ai) 上列出的受支持 AI 模型清单中链接的官方定价来源。

付费方案中，工作区所有者和管理员可以通过手动一次性充值或配置自动充值来为余额充值。

未使用的工作区资金会按月结转，但会在一年后过期。

**一次性充值**

以你选择的金额手动为工作区余额充值。

1. 前往 **Settings → Cloud & AI balance → Top up → One-time to-up.**
2. 输入自定义金额或选择预设选项（最低 \$10，最高 \$1000）。
3. 通过 Stripe 安全完成支付。

**自动充值**

当余额低于 \$10 时自动添加资金，同时通过设置每月收费上限来控制你的月度支出。

1. 前往 **Settings → Cloud & AI balance → Top up → Automatic top-up.**
2. 启用自动充值。
3. 输入充值金额。当余额低于 \$10 时，该金额会被自动扣款并添加到你的工作区余额中（最低 \$10，最高 \$1000）。
4. 设置每月收费上限，以控制每个月最多可以被扣费的总金额（最低 \$10，最高 \$10,000）。
3. 点击 **Save**。\
   当你的余额低于 \$10 且尚未达到每月收费上限时，系统会自动从你的默认支付方式中扣款。

配置自动充值后，你可以在 **Automatic top-up** 页面直观查看你的每月收费情况。每月收费周期会在每个月的第一天 00:00 UTC 重置（会以你的本地时区显示），且与订阅方案的账单周期相互独立。

<div id="alerts-and-notifications">
  ### 警报和通知
</div>

Lovable 会及时通知你，帮助你避免意外的服务中断：

- 向工作区所有者和管理员发送**电子邮件通知**
- 在 Cloud & AI 余额页面和 Cloud 概览中显示**用量警报**

在以下情况下，你的工作区会触发警报：

- 达到包含用量的一定百分比
- 用完所有包含用量
- 达到钱包余额的一定百分比
- 用完钱包中全部资金

如果你的工作区余额耗尽，你的应用将停止运行，直到你再次充值（或等到下个月初免费用量重置）。

要让你的应用持续运行：

- **免费套餐：** 升级到付费套餐以充值。
- **付费套餐：** 可随时为工作区余额充值，或启用自动充值。

<AccordionGroup>
  <Accordion title="我能完全掌控我的每月支出吗？">
    可以。我们设计 Lovable Cloud 的目的就是让你能完全掌控费用：

    * **选项 1：硬性预算（零费用）**
      严格保持在免费额度内。如果超出，项目会自动暂停，你的账单会保持在 $0。
    * **选项 2：手动充值（可控扩容）**
      按你选择的金额为工作区余额充值。你的应用会按需扩容，你只需为超出免费额度的用量付费。
      如果你用完工作区余额，你会收到通知，并且项目会自动暂停。
    * **选项 3：自动充值（无缝扩容）**
      当余额低于 $10 时自动添加资金，但你仍然可以通过设置月度扣费上限来控制每月费用。你的应用在不中断的情况下扩容，你只需为超出免费额度的用量付费。
      如果因为达到月度扣费上限而无法自动充值，当剩余工作区余额用完时，你的项目将暂停。
  </Accordion>

  <Accordion title="我的额度会同时用于 Cloud 和 AI 吗？">
    不会，你的工作区额度是独立的，只在你构建应用时用于对话消息。你可以在 **Settings → Plans &amp; credits** 中查看额度使用情况。

    你可以在 **Settings → Cloud &amp; AI balance** 中查看 Cloud 和 AI 的使用情况。

    * 你的 Cloud 余额用于支付已部署应用的托管费用。
    * 你的 AI 余额用于支付已部署应用内的 AI 功能费用。
  </Accordion>

  <Accordion title="Lovable AI 的费用是多少？">
    使用 Lovable AI 的费用与直接使用 LLM 提供商的费用完全相同。没有任何隐藏收费。要核对费用，请参考我们在 [Lovable AI](https://docs.lovable.dev/features/ai) 中列出的已支持 AI 模型所链接的官方定价来源。
  </Accordion>

  <Accordion title="我目前在使用 Supabase 集成。这次变更对我意味着什么？">
    不用担心，你现有的项目不会有任何变化。你的 Supabase 集成会像现在一样继续正常工作，我们也会持续提供支持。

    * 对于现有项目：可以继续按原样使用 Supabase，无需任何更改。
    * 对于新项目：可以选择使用 Lovable Cloud 或 Supabase。
  </Accordion>

  <Accordion title="在给项目添加 Cloud 之后，我还能改变主意吗？">
    不能。当你将项目连接到 Cloud 后，就无法撤销这一更改，也不能将项目从 Cloud 中断开。
  </Accordion>

  <Accordion title="我可以复刻 Cloud 项目吗？">
    可以，Cloud 项目只能复刻为 Cloud 项目。复刻会在你当前选择的工作区中创建一个可单独编辑的项目副本。与后端相关的数据结构（例如表和 Edge Functions）会被复制过去，但不会包含任何具体数据。对于原始项目中自动生成的密钥，Lovable 会在复刻项目中生成新的密钥；手动添加的密钥不会被复制。

    请注意，如果项目使用的是 Lovable 后端，你无法在复刻项目中将应用恢复到之前的版本。对于仅包含前端、托管在 Cloud 上的项目，可以进行版本回退。
  </Accordion>

  <Accordion title="我可以把 Cloud 项目导出到 Supabase 吗？">
    从技术上讲这是可行的，因为你在 GitHub 中拥有代码，但过程并不简单。我们正在研究如何让这一过程更容易。更多信息请参见 [Self-hosting: Run your Lovable Cloud project anywhere](https://docs.lovable.dev/tips-tricks/self-hosting)。
  </Accordion>

  <Accordion title="我怎么知道哪些项目产生了最多的用量？">
    你可以前往 **Settings → Cloud &amp; AI balance**，按项目查看 Cloud 和 AI 的用量明细。

    <Note>
      目前列表会包含所有项目，即便是没有使用 Lovable Cloud 或 AI 的项目。没有用量的项目会始终显示 $0.00。
    </Note>
  </Accordion>

  <Accordion title="我可以把项目从 Supabase 迁移到 Cloud 吗？">
    目前尚不支持从 Supabase 迁移到 Cloud。如果你的项目已经连接到 Supabase，我们建议继续使用，因为我们会在未来持续支持 Supabase。
  </Accordion>

  <Accordion title="我的 Cloud 数据没有加载，我该怎么办？">
    如果你在较小规格的 Cloud 项目实例上承载了过多数据，就可能出现这种情况。要解决这个问题，请前往 **Cloud 选项卡 → Overview → Advanced settings**，然后选择更大的实例规格。切换到更大的实例后，等待几分钟完成变更，数据应会重新加载，你的 Cloud 项目也应会恢复正常。请记住，更大的实例规格会带来更高的 Cloud 用量和更高的 Cloud 使用成本。
  </Accordion>
</AccordionGroup>

Lovable offre integrazioni native e fluide per esportare il codice e aggiungere un backend. Puoi anche integrare un&#39;API esterna se vuoi aggiungere funzionalità specifiche al tuo sito web.

## Che cos&#39;è un&#39;API?

Un&#39;API è un modo che consente alle applicazioni software di comunicare e condividere dati tra loro.
API diverse offrono funzionalità differenti, come l&#39;invio di email o l&#39;elaborazione di pagamenti.
Puoi approfondire il ruolo e la potenza delle API in [questo articolo sul nostro blog](https://lovable.dev/blog/2025-01-28-the-power-of-apis).

<div id="using-any-api-with-lovable">
  ## Utilizzare qualsiasi API con Lovable
</div>

Tutte le altre API invocabili possono comunque essere integrate, ma spesso richiedono contesto e istruzioni aggiuntive.

Potresti avere:

* **Sistemi interni con API personalizzate**\
  Potrebbero essere servizi che il tuo team ha creato (ad es. sistemi di inventario, pipeline di analisi, strumenti interni) che espongono endpoint REST o GraphQL ma che non sono documentati pubblicamente o ampiamente conosciuti.
  Vuoi che Lovable legga o invii dati a questi sistemi per automatizzare flussi di lavoro o alimentare interfacce utente.
* **API di terze parti che Lovable ancora non supporta**\
  Magari usi un prodotto SaaS di nicchia, un provider di dati specializzato o un&#39;integrazione con un partner che Lovable non conosce ancora nativamente.
  Anche se non è ufficialmente supportata, vuoi comunque che Lovable si connetta, effettui chiamate e lavori con i dati.

In entrambi i casi, la sfida è la stessa: per impostazione predefinita Lovable non “conosce” l’API. Ma con una buona documentazione e una configurazione corretta dell’autenticazione, può comunque usarla in modo efficace.
In ogni caso, l’approccio è lo stesso: **Lovable può comunque connettersi**. Ecco come.

<Steps>
  <Step title="Fornisci a Lovable il contesto di cui ha bisogno">
    Lovable funziona al meglio quando comprende l’API. Puoi fornire:

    * **Specifica OpenAPI** (se disponibile)
    * **Documentazione o esempi copiati e incollati** (in particolare endpoint e richieste/risposte di esempio)

    Più la documentazione è chiara, meglio Lovable può ragionare su come chiamare l’API.
  </Step>

  <Step title="Gestisci l'autenticazione in modo sicuro">
    Per le API che richiedono autenticazione:

    **Opzione 1: Usa le Supabase Edge Functions come proxy**

    * Archivia i Segreti in modo sicuro su Supabase
    * Lascia che Lovable chiami l’endpoint dell’API tramite la Funzione edge (non direttamente l’API)

    **Opzione 2: Implementa l&#39;integrazione con l&#39;API direttamente nel tuo Progetto Lovable**

    * Esponi un endpoint di autenticazione (ad es. per scambiare credenziali con un token o una sessione)
    * Consenti a Lovable di chiamare quell’endpoint di autenticazione come parte del flusso di integrazione
    * Includi token, header o informazioni di sessione nelle chiamate API successive
  </Step>
</Steps>
